　在學習安裝 Linux 之前，有一些觀念您必須要先建立起來，包括 Linux 的目錄結構、檔案系統的認識、裝置名稱及分割區 (partition) 等等。或許對於第一次接觸 Linux 的您來說，有部分內容可能會看得有點吃力，不過沒關係，在往後的章節裡邊，對這些相關的概念，都還會有更進一步的說明。

2.1 安裝 Linux 前需有的認知

　這裡是要跟大家簡單的介紹安裝前需先了解的事項，而有了這些初步的認識之後，您在安裝 Linux 時，才不會有一種知其所以、卻不知其所以然的感覺。

2.1.1 Linux 的目錄結構

　Linux 的目錄結構，是一個階層式的樹狀結構，最上層為根目錄 (root directory)，以 " / " 來表示，至於在根目錄下還有一些主要目錄，當然設置這些不同的目錄是有其個別的用途啦。

　在 Linux 檔案系統裡頭的目錄配置，是符合 FHS 的標準規範的。FHS (Filesystem Hierarchy Standard) 主要是針對兩層目錄來定義，第一層是定義根目錄的下一層目錄要存放些什麼資料，比如 /lib 是安置函式庫的地方，/bin 及 /sbin 是放置可執行程式，/etc 是存放設定檔等等 ﹔第二層則是定義 /usr 及 /var 的下一層目錄又要存放些什麼資料，比如 /usr/src 是存放原始碼，/var/log 是存放紀錄檔。

　只要您所使用的 Linux Distribution 是符合 FHS 標準的話，那相關檔案所存放的位置應該不難找到。至於 FHS 的詳細解說可以參考：http://www.pathname.com/fhs/ 。

˙/usr/bin：與 /bin 的意思差不多，比較屬於使用者導向。
˙/usr/sbin：與 /sbin 差不多意思。
˙/usr/src：存放原始碼的地方，比如核心原始碼就放在這裡邊。
˙/usr/share：提供給使用者所分享的文件就放在這裡。比如：
　--- /usr/share/doc：存放很多的套件相關說明文件。
　--- /usr/share/man：man page 資料存放處。
˙/usr/local：自行下載軟體來安裝時的預設安裝位置。
˙/usr/lib：函式庫的存放位置。

這裡說明一下 /bin、/sbin 及 /usr/bin、/usr/sbin 的差別。/sbin 目錄主要是存放一些開機過程、系統復原或系統緊急修復時，需使用到的程式 ﹔而 /bin 則是包含一些當沒有其他檔案系統被掛載時 (比如單人模式)，可以被執行的程式。至於除了這兩個目錄外的其他一般程式，就放在 /usr/bin 與 /usr/sbin 了。

　如果您是第一次接觸 Linux，會覺得說怎麼光是它的目錄結構就那麼複雜，不免心理感到有點慌，其實我們上面所列出每個目錄的功用，乍看之下是很多沒錯，但千萬不要去死記，因為隨著您學習時間的拉長，且對系統整體的概念愈來愈清晰時，自然而然的就會了解這些目錄是做什麼用的。

2.1.2 檔案系統及裝置檔案的認識

　何謂檔案系統 ? 就是管理檔案的系統囉。在電腦系統中，資料是存放在儲存裝置中的，至於這些資料到底要存放在裝置裡的哪個位置，就必須要有一套管理這些檔案資料的系統來做處理，此即檔案系統是也。

　 如您曾經安裝過 Windows 的話，應該很清楚的知道，在安裝之前必須先對分割區執行格式化的動作，而這個動作就是在製造一個檔案系統給這個分割區來使用，這樣待會兒在安裝的過程，才能順利把檔案資料寫入此分割區的檔案系統中。

　Windows 有 Windows 的檔案系統，如 VFAT、NTFS 等，那 Linux 當然也有它專用的檔案系統，如 ext2、ext3、reiserfs、JFS … 等等，由於這兩種作業系統所使用的檔案系統架構完全不相同，所以資料存取的模式自然也就不一樣啦。

　受許多玩家青睞的 RedHat，其預設的檔案系統為 ext3，而今天要學的 SuSE，則是使用 ReiserFS，當然您想讓 SuSE 使用 ext3 也是可行的。至於其他有關檔案系統的相關課題，等到第五章再做進一步的說明。

　再來談談裝置檔案部分。在 Linux 系統中，會把周邊設備及裝置當成檔案來看待，並將這些裝置檔案存放在 /dev 目錄下，這部分於先前的目錄結構中已有提及，而這裡主要是針對與磁碟相關的裝置來做探討。

　傳統的主機板上有兩個 IDE Controller，每個 IDE Controller 可接兩個裝置，而在第一個 IDE Controller (Primary) 上的第一個裝置 (Master)，其檔案名稱為 hda，第二個裝置 (Slave)，其檔案名稱為 hdb ﹔至於第二個 IDE Controller (Secondary) 上的第一個裝置 (Master)，其檔案名稱為 hdc，第二個裝置 (Slave) 就為 hdd。

　在裝置檔案名稱後面，若接一個數字，則此數字代表的就是 partition number (分割區代號)，比如 /dev/hda3 是表示第一個 IDE Controller 上的第一顆硬碟中的第三個分割區，hdd5 是表示第二個 IDE Controller 上的第二顆硬碟中的第五個分割區。

　如果您不是使用傳統的 IDE 硬碟，而是使用 SCSI 的硬碟，則其裝置檔案名稱為 sda、sdb、… 等，那分割區自然就是以 sda1、sda2、… 來表示囉 ! 至於目前的主流 SATA 硬碟，通常其裝置檔案名稱也是以 sda、sdb、… 來命名居多。

Tips：Linux 的檔案系統與 Windows 完全不相同，在 Windows 底下，每個分割區或裝置都有其各自的磁碟機代號，以分割區來說，如您硬碟上的 hda1 是安裝 Win2000，hda2 是用來存放一些備份資料，那麼在 Windows 中這兩個分割區分別是以 C 磁碟機 ( C: ) 及 D 磁碟機 ( D: ) 來表示，但在 Linux 裡，並沒有所謂的磁碟槽概念。

2.1.3 磁碟分割與掛載

　這一小節一開始，先來聊聊磁碟的構造。一顆硬碟中會有好幾片的磁碟盤，每個磁碟盤會有兩面，每一面都有其各自的讀寫磁頭 (Head)。而磁碟盤由內到外一圈一圈的同心圓，就叫做磁軌 (Track) ﹔磁碟盤最外圍的磁軌為第零軌，依序往圓心累計編號。再來每一片磁碟盤的相同編號磁軌所組成的一組磁軌就叫做磁柱 (Cylinder)，此磁柱即是磁碟分割的最小計算單位。接著把磁碟盤從圓心開始，分別畫出無數條半徑，如此便分割成一片一片的扇形區域，像這樣每兩條半徑所分割出來的區域，就稱為磁區 (Sector)，每一個磁區的大小通常為 512 bytes。

　有了以上的觀念後，那麼通常在計算一顆硬碟容量時，即可使用以下的公式做計算：

　　　 　　　　磁柱數(每個表面的磁軌數) x磁頭數(可讀寫的表面) x磁區數 x 每個磁區大小

　熟悉了磁碟構造後，以下開始說明分割區的部分。磁碟分割區可區分為主要分割區 (primary partition) 及延伸分割區 (extended partition)，而一顆硬碟上頭，最多只能有四個主要分割區，其 partition number 為 1~ 4，如以 hda 來說的話，就是 hda1、hda2、hda3、hda4。所以在規劃磁碟分割的時候，如果分割區數小於等於四個時，則全部切割成主分割是沒問題的，但若分割區數比四個還多時，那就必須要有延伸分割區的存在了。延伸分割區的 partition number 也是 1~ 4，不過不能與 primary partition 重複，以 hda 來說，當所規劃的主分割為 hda1 及 hda2 時，則延伸分割區就只能為 hda3 或 hda4 了。

　您不能對延伸分割區本身製造檔案系統，因此需要在延伸分割區之中，再去切割成幾個邏輯分割區 (logical partition) 出來，然後才能對此邏輯分割區進行格式化的動作，而邏輯分割區的 partition number 是從 5 開始的。

　了解了分割區的概念後，或許您會想知道整顆硬碟到底可切割成多少個 partition，一般來說，如使用的是 IDE 硬碟，其 partition number 可達 63，若是 SATA 或 SCSI 硬碟，則其 partition number 只能達 15 而已。

　最後談談掛載的概念。在 Linux 中，所有被 Linux kernel 所支援的檔案系統，都必須被掛載到根目錄下的某個目錄或子目錄當中，才能夠被存取使用，被掛載的這個目錄就稱為掛載點 (mount-point)，而將檔案系統掛載至掛載點的動作便稱為掛載 (mount )。比如您要存取 cdrom 或 floppy 時，就必需先將它們掛載至根目錄下 (如 /media/cdrom、/mnt/floppy) 才行 ﹔那對磁碟分割區來說也是一樣的道理，您必須要把分割區掛載至根目錄下之後，才能存取該 partition 的內容。

　在 Linux 的目錄結構中，已了解到最上層為根目錄，所以想當然爾，在啟動 Linux 作業系統的過程中，第一個需被掛載的 partition 就是 root partition。

2.2 安裝 SuSE Linux

　筆者通常會建議初學者在安裝 Linux 時，能使用單純一點的環境，也就是使用主機上唯一的一顆硬碟，來專門安裝 Linux，除非您熟悉多重開機的做法，否則先不要嘗試。不過若您迫切想學 Linux，但不會做多重開機，而手邊也沒多餘的電腦可用，那只好利用 vmware 來應急一下，關於這一部份在 2.3 節會有相關說明。

　另外了解一下安裝 SLES10 (SuSE Linux Enterprise Server 10) 的最低系統需求，首先記憶體部分建議是 512 MB，供軟體安裝的最小硬碟空間是 750 MB，至於支援的處理器平台有 x86、AMD64、Intel EM64T、Itanium 處理器系列等等。

2.2.1 規劃磁碟空間配置

　一般而言，安裝 Linux 作業系統最少要有兩個 partition，分別為 / 及 swap。swap 是所謂的交換區或稱為虛擬記憶體，其主要作用是當實體記憶體不足時，會先將一些平常使用頻率較低的程式暫存至 swap 內，這樣才可讓出更多的實體記憶體來給真正要執行的程式使用，而等程式執行完畢後，再把那些剛剛放在 swap 裡的程式放回記憶體內。

　等一下安裝 Linux 時，會練習切割三個分割區，來分別提供給 /、/home 及 swap 使用。將來要是您對系統更加熟識時，可以練習劃分更多的 partition，比如可讓 /、/boot、/usr、/var、/home、/srv、/tmp 目錄都擁有自己的 partition。那劃分那麼多 partition 要幹嘛 ? 這就說來話長了，等您對 Linux 有更進一步的認識時，自然就能體會它的好處了。

　再來談到磁碟空間的配置。如果依照筆者的需求，所需安裝軟體大約 2G 上下，而硬碟空間有 5G 可使用，因此等一下會給 root partition 4G，/home partition 300M，swap partition 也是 300M，還剩下的空間，將來可提供做磁碟分割練習用。如果您的磁碟空間不像筆者那麼少的話，則可以適時的將這些分割區調整大一些。

2.3 使用 VMware 來安裝 SuSE Linux

　VMware (Virtual Machine Software) 是一套虛擬機器的軟體，它可以讓您在現有的 PC ( 此 PC 稱為 Host PC ) 中再模擬出一台或一台以上的 PC ( 此 PC 稱為 Guest PC )，說得白話一點，就是可以讓您在現行的作業系統下，再去安裝其他作業系統的意思，這對電腦數不夠的您來說，可不失為一良好的解決方式。不過您主機本身的等級可是要高一點噢，因為 Guest PC 所需使用的資源，可是從 Host PC 而來的，所以您主機本身必須要有足夠的記憶體與硬碟空間才行。

　由於筆者的構想，是希望能在 Windows 中去跑 Linux，因此需要去下載 for Windows 的 VMware 軟體。當然如果將來有一天，您想在 Linux 中去跑 Linux 或 Windows，那就須下載 for Linux 的 VMware 囉。

6. 選擇所要使用的網路型態：

這裡稍微解釋一下常見的兩種網路連接方式：

˙ Use bridged networking：
  如果外部網路能提供一個 IP 位址給 Guest PC 的話，就選擇這個項目。比如平常您 Host PC 是透過 ADSL 或 DHCP 來取得 IP，現在也希望 Guest PC 能使用相同方式來取得時，那選此項目就對了。
˙ Use network address translation(NAT) ：
  如果 Guest PC 無法向外部網路取得 IP 位址時，可選擇這個項目來讓 Guest PC 透過 Host PC 連上 Internet。

  是否要事先分配所指定的虛擬磁碟空間大小。當您磁碟空間足夠時，可以考慮勾選此項目，且如此一來，您虛擬磁碟運作起來也會比較有效率。
˙ Split disk into 2 GB files：
  當您 Windows 是使用 VFAT 檔案系統時，必須要把這個項目勾選起來，以免超過其檔案大小的限制。比如 FAT 16 限定檔案大小為 2GB，FAT 32 則為 4GB。那如果使用 NTFS 的話，則毋須擔心，因其檔案大小限制是 4TB。

2.4 YaST 工具簡介

　SuSE Linux 裡頭，有一個很強悍的工具叫做 YaST(Yet another Setup Tool)，舉凡系統的安裝過程及進入到系統後的設定，幾乎都能夠靠他來搞定，且不論是在文字介面或者圖形介面下，皆能使用這個工具來操作。不過話雖如此，可千萬不要過度依賴他，尤其是初學者。如果打從您學習 SuSE 開始，就一直使用 YaST 來設定系統，那將會造成對相關指令及設定檔的不熟悉，甚至對於系統上的一些重要觀念也會忽略掉，所以建議玩 SuSE 的朋友，把 YaST 當成一個輔助工具來看待就好。另外提醒一下，當使用 YaST 時，可是要以 root 的身分來操作噢。
