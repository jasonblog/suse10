　這一章會介紹很多 GNU 及 UNIX 中常用到的一些基本指令，諸如檔案目錄瀏覽的指令、支援正規表示法的指令、vi 文字編輯器、 echo 及 export 指令等等 …，當然與這些指令相關的系統觀念，我們也會一併做個說明，像標準輸入輸出、環境變數、正規表示式與萬用字元等，而這些內容對初次探索 Linux 的您來說，可說是非常非常的重要，請您務必要在這上頭多花一點時間。

　不過在此之前，您可能要先學會一些系統的基本操作，之後才能繼續進行下一步驟的學習。

3.1 系統入門的基本操作

　在這一節裡面，是要教大家能對 Linux 的系統有一個初步的認識，以事先熟悉一下我們的操作環境。

3.1.1 shell 概述

　初次接觸 Linux 的朋友，對 shell 這個 Unix 術語可不能不認識啊，因為將來您在系統上所做的任何操作，都跟它脫離不了關係。話不多說，馬上了解 shell 去。

　簡單來說，shell 就是一個使用者的操作介面，它可讓您透過螢幕和鍵盤來跟作業系統的核心做溝通。由於電腦和我們人類所使用的語言不同，因此需要透過 shell 來讓核心了解使用者到底想做什麼事。以下是核心、shell 及使用者之間的關聯圖：　

　舉例來說，當使用者在 shell 下輸入 date 指令時，由於核心認得的是二進位的程式碼，它並不認識 date 是什麼東西，因此需要透過 shell 來解析成核心所認得的二進位程式碼，如此工作才有辦法順利進行 ﹔而在核心做出回應時，就依反方向進行，當然此時還是會經過 shell 來把系統所提供的訊息再解析成我們人類所看得懂的語言，最後才能於螢幕上看到執行的結果。從以上說明可以清楚知道，其實 shell 就是一個命令解析器 (command interpreter) 啦。

　第一個 Unix 系統所使用的 shell 為 Steve Bourne 所寫，稱之為 sh，在 sh 出現後，一些人相繼發展出各種不同的 shell，而大部分的 distribution 預設所使用的 shell 為 Bourne Again SHell，就是所謂的 bash，是依 sh 發展而來的，像我們目前所要學習的 SuSE Linux 也是使用這個 bash。而在 Linux 文字模式下，除了 bash 以外，還有多種的 shell 可供選擇，如 sh、ash、ksh、zsh、csh、tcsh、… 等，參考一下 /etc/shells 檔案就知道了。

3.1.2 虛擬控制台 (Virtual Console)

　Linux 是一套多人多工的作業系統，在其主控台 (console) 下，提供了預設的六個文字介面的虛擬主控台及一個圖形介面 (X-WINDOW ) 主控台給使用者來操作。

　如您於開機後所看到的是圖形介面的登入畫面，可以按下 Ctrl - Alt - F1 ~ F6 (tty1 ~ tty6) 來切換至其他的文字介面的主控台。又如您目前已在第一個文字介面主控台時，則可使用 Alt - F2 ~ F6 切換成其他文字介面主控台，或 Alt - F 7 回到圖形介面主控台。所以從文字介面的 virtual console 切換成其他的 virtual console 時，可以只使用 Alt - Fn (n 表示第幾個 virtual console) 來切換，當然您要使用 Ctrl - Alt - Fn 也是可以啦，不過從圖形介面要切換至其他的 virtual console 則需使用 Ctrl - Alt - Fn 切換。

3.1.3 首次登入與提示號樣式

　了解如何切換 virtual console 之後，接著請您切換至第一個文字介面終端機 (Ctrl - Alt - F1)，然後會看到 login 的提示號，要求您輸入登入帳號，此時請您先以管理者 root 的身分進行登入，再來會要求您輸入密碼，請輸入當初安裝 SuSE Linux 時所設定的 root 密碼 (輸入密碼時，螢幕上是不會留下任何紀錄的)，若認證無誤的話，就可以看到 root 的提示號樣式，如以下所示：

suse login : root
password :
Last login: Wed Jan 3 17:02:12 2007
suse:~ # _

　SuSE 預設 root 的提示號中，":" 左手邊是您的主機名稱 (hostname)，也就是 suse，那右手邊就是當前的工作目錄位置 (work directory)，"~" 代表的是使用者個人的家目錄，也就是說當使用者登入後，其預設的工作目錄就是自己的家目錄 ﹔最後則是用一個 "#" 符號來表示是管理者登入。

　不過這個提示號樣式可以修改，等後面提及環境變數時再來談。至此我們已經可以開始輸入一些指令來執行工作了。

3.1.4 下達指令的基本語法

　既然已經看到了命令列上所提供的提示號，那接著當然就可以於提示號下輸入指令囉，因此對下達指令的基本語法就要有所了解，如下所示：

suse:~ # command　<options>　<arguments>

　大部分的指令都可以搭配一些參數 (options) 來使用，參數後可以接一些引數 (arguments)，不過指令真正的用法還是要看設計師當初如何設計這支程式來決定如何下達這個指令的。

　這裡要注意的是 command、options、arguments 之間最少要有一個空格，若您的 command、options 或 options、arguments 之間忘了空格，而是直接連在一起的話，那會造成 shell 解析上的錯誤，這是初學者易犯的毛病。

↑、↓ 假使之前曾經輸入了一個很長的指令敘述，現在想要再執行一次，則可重複按下↑鍵去叫出之前所執行過的指令敘述，不過萬一按太快而過頭了，那麼利用↓鍵即可反方向搜尋。

3.1.6 新增使用者帳號

　首先先來練習新增使用者帳號，請您以管理者 root 的身分來執行這項工作：

suse:~ # useradd -m barry
suse:~ # passwd barry
Changing password for barry.
New password:
Re-enter new password:
Password changed
suse:~ #
　
　useradd 及 passwd 就是兩個指令，-m 為 options，barry 當然就是 argument 囉，而這裡的 argument 是一個您即將新增的帳號。

　我們是先使用 useradd 指令來新增使用者 barry 的帳號，加上 -m 參數是為了讓 barry 有家目錄可用，輸入完畢請按下 Enter 鍵。再來是使用 passwd 指令來幫 barry 建立(修改)密碼。請注意 passwd 指令之後需指定剛剛建立起來的那個使用者帳號，若指令之後忘了指定帳號，則會變成修改 root 自己的密碼。至於這兩個指令的詳細用法，在下一章的 [系統管理工作] 會有詳細說明。

　Linux 是一套非常講究安全的作業系統，之所以建立一般使用者帳號，是希望大家平時盡量養成用一般使用者登入的習慣，除非要對系統作相關設定時才使用 root 身分來登入。

　好了，使用者 barry 既然已經被建立起來了，就請您按下 Alt-F2 切換到第二個文字介面終端機，並以 barry 的帳號密碼來登入，認證成功後，您應該可以看到一般使用者的提示號像這樣：

barry@suse: ~ >

　或許您又有疑問，怎麼一般使用者跟 root 的提示號樣子長得不一樣，還是老話一句，等到環境變數時再來解釋。不過這裡您應該要知道在 "@" 前面的 barry，就是使用者 barry 的登入帳號，至於最後方的 ">"，只是個有別於管理者樣式的一個符號罷了。

　最後請讀者再使用相同的方式去新增另一個使用者 mary，因等一下的指令練習會使用到。

3.1.7 擅用 manual page 線上說明手冊

　Linux 系統上存在著很多的說明文件，而在您學習的過程中，可能會想要了解某些指令的詳細用法或者某些檔案設定格式的相關說明，那這時候就可以善加利用 man 這個指令。比如剛剛在新增帳號時用了 useradd 及 passwd 指令，如想對這兩個指令有更進一步的認識，則可在提示號下執行 "man useradd" 或 "man passwd" 去了解這兩個指令的詳細用法。

　而在 man page 下的操作也很簡單，您可以使用 [Page Up]、[Page Down] 去翻頁，也可使用 "/string" 去快速搜尋符合這個關鍵字串的位置 (string 為您欲搜尋的關鍵字串)，最後按下 q 就可以退出 man page。至於詳細用法，請參考本章 3.8 節的說明。

3.1.8 登出及關機

　既然學會了登入，再來當然要知道如何登出囉，您可以在命令列 (command line) 上執行 exit 或 logout 指令，不然使用 Ctrl-D 的組合鍵也行，這就請各位自行練習。

　最後介紹如何關機，首先您必須以 root 的身分來執行以下指令的其中一個：

suse:~ # init 0
suse:~ # halt
suse:~ # shutdown -h now
suse:~ # poweroff

　若要重新開機，則可以這麼做：

suse:~ # init 6
suse:~ # reboot
suse:~ # shutdown -r now

3.2 系統入門的基本指令

　了解了基本操作之後，再來就開始介紹基本的入門指令，這包括目錄切換及檔案目錄瀏覽指令的使用。

指令 cd

　改變目前工作目錄位置的指令。

指令語法：
cd [option] [dir]

範例說明：
barry@suse:~> cd /var
barry@suse:/var>
# 切換工作目錄至 /var。當在命令列上所輸入的指令或其後的引數路徑，是從根目錄開始
# 指定起，就稱為絕對路徑(absolute path)，反過來說若非以根目錄指定起，即為
# 相對路徑 (related path)。
barry@suse:/var> cd
barry@suse:~>
# 只下達 cd 指令，其後未指定所要切換的目錄時，則表示要切換至自己的家目錄。
# 另外也可執行 "cd ~" 來達同樣的效果 ﹔~ 就是該使用者自己的家目錄。
barry@suse:~> cd ~mary
barry@suse:/home/mary>
# 表示要切換至使用者 mary 的家目錄。
barry@suse:/home/mary> cd /tmp
barry@suse:/tmp> cd -
/home/mary
barry@suse:/home/mary>
# "cd -" 表示切回前一個目錄，而 /tmp 的前一個目錄就是 /home/mary。
barry@suse:/home/mary> cd ..
barry@suse:/home>
# ".." 是表示上一層目錄，而 mary 的上一層目錄為 /home。
barry@suse:/home> cd /var
barry@suse:/var> cd log
barry@suse:/var/log>
# 由於欲切換的目錄 log 就位於目前這個工作目錄 /var 之下， 因此直接輸入 log 就行了。
barry@suse:/var/log> cd ./samba
barry@suse:/var/log/samba>
# "." 表示當前的工作目錄 /var/log，"./samba" 就代表著是 /var/log/samba。
barry@suse:/var/log/samba> cd ~/bin
barry@suse:~/bin>
# 切換至自己家目錄下的 bin 目錄。
barry@suse:~/bin> cd ../public_html
barry@suse:~/public_html>
# 切換至上一層目錄下的 public_html 目錄。

barry@suse:~/public_html> cd -P /var/mail
barry@suse:/var/spool/mail>
 # /var 目錄下的 mail 為一符號連結檔，連結至 /var/spool/mail，而當 cd 指令
# 加上 "-P" 參數時，就可以切換至實際的目錄位置。
# 若您要了解目前所在的目錄位置，可使用 pwd 指令：
barry@suse:~> pwd
/home/barry

指令 ls

　這是用來瀏覽目錄內容的指令。

指令語法：
ls [OPTION]... [FILE]...

參數說明：
-a
 列出所有的檔案，包含隱藏檔。
-A 列出所有的檔案，包含隱藏檔，但 . 與 .. 這兩個目錄除外。
-d 只列出目錄本身的紀錄，不包含目錄的內容。
-h 檔案大小的單位，是以使用者容易閱讀的方式做顯示 ﹔需搭配 -l 使用。
-i 列出每一個檔案的 inode number。
-l 以長格式來顯示。
-R
 遞迴地列出此目錄樹系下的所有內容，而非只列出下一層的內容。
-r
 使用 ls 指令來列出目錄內容時，其所顯示的名稱，預設是按照數字、英文字母大小寫來做排序，而當您加上 "-r" 參數後，便會反向做排序。
-t 依照檔案的修改時間做排序，較新的時間排在前。
-u 依照檔案的存取時間做排序，較新的時間排在前。此參數若與 -lt 一起使用，會依檔案的存取時間做排序，如只與 -l 搭配使用，則會依檔案名稱來做排序。

範例說明：
barry@suse:/> ls
bin　 dev　home　media　opt　 root　srv　　　　sys　usr
boot　etc　lib 　mnt　　proc　sbin　subdomain　tmp　var
# ls 指令之後未指定欲瀏覽的目錄，則預設會列出當前工作目錄下的內容。
# 一般來說，如名稱是藍色，其代表的是目錄，白色的則是檔案，天藍色的是符號連結檔。

barry@suse:~> ls -l /var
drwxr-xr-x　5 root root 152　Oct　7 23:20 X11R6
drwxr-xr-x　8 root root 224　Dec 21 23:49 adm
drwxr-xr-x 12 root root 304　Oct　7 23:26 cache
# 列出 /var 目錄的內容，並以長格式顯示。
barry@suse:~> ls -l /etc/passwd  
-rw-r--r--　1 root root 1582 Jan 3 17:24 /etc/passwd
 # 列出 /etc/passwd 檔案本身的長格式。

barry@suse:~> ls -ld /etc
drwxr-xr-x 81 root root 7288 Jan 4 13:15 /etc
# 只列出 /etc 目錄本身的長格式，不包含其下的內容。

barry@suse:~> ls -a
. 　　　　　　 .bashrc　　 .dvipsrc　.fonts　　　 .profile　 　.viminfo　
.. 　　　　　　bin 　　　　.emacs　　.kermrc 　　 public_html　.xcoralrc　
.bash_history　Documents　 .exrc 　　.muttrc　　　.urlview 　　.xemacs 　
# 列出目錄下的所有檔案。您看到很多名稱前面有個點的檔案，那就是隱藏檔啦。
# 而列出內容中還看到 . 及 ..，其代表的就是現在這個目錄及上一層目錄，如不想
# 見到這兩個固定會顯示的內容，執行 "ls -A" 即可。
barry@suse:~> ls -ldi /var
2 drwxr-xr-x 16 root root 392 Oct 7 23:08 /var
 # 顯示 /var 本身的長格式。最前面那個 2，是 /var 目錄的 inode number。
# inode 的觀念，請參考檔案系統那一章。

指令 cat

　這是瀏覽檔案內容的指令。

指令語法：
cat [OPTION] [FILE]...

範例說明：
barry@suse:~> cat /etc/passwd /etc/fstab
# 瀏覽 passwd 及 fstab 兩個檔案的內容。
barry@suse:~> cat -n /etc/filesystems
　　1 vfat
　　2 hfs
　　3 minix
　　4 reiserfs
　　5 *
# 加上 -n ( number ) 參數，就會將檔案內容的每一行，包括空白列，都附上編號列出來。
# 另外有個很類似但不完全相同的指令 nl，請參考以下範例：
 barry@suse:~> nl file
　　1 hello,everybody.
　　2 i am barry.

　　3 how are you?
barry@suse:~> cat -n file
　　1 hello,everybody.
　　2 i am barry.
　　3
　　4 how are you?
# 這樣應該已經看出來 nl 跟 cat -n 之間的差別吧。

指令 tac

　與 cat 指令的功能一樣，只是把輸出的內容反向做排序而已。

範例說明：
barry@suse:~> cat /etc/filesystems
vfat
hfs
minix
reiserfs
*

barry@suse:~> tac /etc/filesystems
*
reiserfs
minix
hfs
vfat

　看出不同點了嗎，cat 輸出內容的第一行變成了 tac 輸出內容的最後一行了。


指令 more、less

　當檔案內容太多時，可以使用這兩個指令讓我們慢慢翻頁看。

範例說明：
barry@suse:~> more /etc/termcap
# 用 more 來瀏覽檔案時，可使用空白鍵來往下翻一頁，使用 b key 來往回翻一頁。
# 若要離開 more，按下 q 即可退出。
barry@suse:~> less /etc/termcap
# 與 more 一樣可使用空白鍵或 b key 來翻頁，另外也可搭配 [Page Up] 及 [Page Down]
# 使用，不過其一次只能翻半頁噢。至於離開 less 時，也是使用 q 來退出。
　
　當檔案內容很多時，想搜尋某個關鍵字所在的位置時，可於進入 more 或 less 畫面後先輸入 " / "，然後緊接著就輸入您要搜尋的字串，完成後按下 Enter 鍵就可以很輕易的找到了。

　以上面的範例為例，在您開啟 /etc/termcap 後，請輸入 "/compatible"，然後按下 Enter 鍵試看看，如成功搜尋到之後，再請輸入 "n" 來尋找下一個 compatible ﹔若要回到上一個 compatible 就輸入 "N"。

　另外在 more 或 less 畫面下，想要進入到 vi 的 command mode，則可以輸入 "v" 來進入，退回原 more 或 less 畫面則輸入 ":q"。至於 vi 的相關說明請參考 3-6 節。


3.3 資料流重導向及管線

3.3.1 資料流重導向

　資料流 (stream) 就是指資料的流向，話說當我們要執行指令時，可能會接受一些資料的輸入，並將程式處理後的結果做個輸出，至於要從哪裡讀進所輸入的資料，又要將指令執行結果輸出到哪裡，這就需要先認識三種基本的資料流：
  標準輸入：standard input (stdin)
    預設是讓程式從鍵盤來取得資料，其 FD 代碼為 0。
     
  標準輸出：standard output (stdout)
    預設是將程式執行的結果輸出到螢幕，其 FD 為 1。
     
  標準錯誤輸出：standard error (stderr)
    預設是將程式執行的錯誤結果輸出到螢幕，其 FD 為 2。

　註：FD (File Descriptor)是用來告訴程式要從哪裡讀進資料，又要將結果輸出到哪裡的意思。

　由以上解說不難發現，一般所執行的程式都會把這三種標準 I/O 通道，連接至控制台的輸入與輸出裝置，而輸入裝置就是指鍵盤，輸出裝置則是螢幕。不過話雖如此，您還是可以使用一些重導符號來將其轉向至其他地方去，比如可將 stdout 重導至檔案或儲存媒體中，也可將 stdin 從原本的鍵盤轉向至檔案。

　以下就列舉幾個重導符號的使用範例。

指令 > 檔案

　將指令執行的結果 (stdout) 轉向輸出至檔案之意。

範例說明：

barry@suse:~> cat /etc/passwd > barryfile
# 將 cat 指令的 stdout 轉向輸出至 barryfile 檔案內，此時螢幕上是看不到任何輸出訊息的。
 # 這裡有個地方要注意，如果 barryfile 是一個不存在的檔案，則會先幫您建立起這個檔案，
# 然後再把 passwd 的內容放進來，但若 barryfile 是一個已經存在且有內容的檔案，則會先
# 清空檔案內容後，再把 passwd 的資料放進來，此時若您 barryfile 裡頭有存放一些重要
# 的資料時，當然就會被覆蓋掉啦。
# 為了避免這種事情發生，您可事先執行：
barry@suse:~> set -o noclobber

# 然後做個測試：
barry@suse:~> cat /etc/fstab > barryfile
-bash: barryfile: cannot overwrite existing file

# 如果此時要強制覆寫的話，可使用 >|：
barry@suse:~> cat /etc/fstab >| barryfile

# 欲取消此功能，則執行如下：
barry@suse:~> set +o noclobber

# 再來看個有趣的範例：
barry@suse:~> cat barryfile > barryfile
cat: barryfile: input file is output file
# 執行完畢後，barryfile 會變成一個空檔案 ﹔想想第一個範例所提及的觀念就知道為何了。 

barry@suse:~> cat
my name is barry.　←　從鍵盤所做的輸入。
my name is barry.　←　cat 讀入後，即把結果輸出至螢幕。
# cat 後面若沒接引數，就會從 stdin 讀取資料，所以只執行 cat 指令時，shell 就會等待我們
# 從鍵盤做輸入，然後再把所讀入的資料給做 stdout，此時當然就是輸出到螢幕囉。
# 若要終止輸入時，按下 Ctrl- D (送出EOF 給 Shell)就行了。

barry@suse:~> cat > file
# cat 從 stdin ( keyboard ) 讀取資料後，轉向輸出至 file。

barry@suse:~> cat /etc/passwd noexistfile 1> file1 2> file2
# 將 cat 指令執行結果，其 stdout 輸出至 file1，stderr 輸出至 file2。
# FD 代碼為 1 是預設值，可省略不寫。
# 另外請留意 FD 與 ">" 之間不能留有空格，不然 shell 會把 FD 代碼當成一個檔案來看待。

barry@suse:~> cat /etc/passwd noexistfile > file3
# 將 stdout 輸出至 file3，stderr 輸出至螢幕。

barry@suse:~> cat /etc/passwd noexistfile > file4 2>&1
# 2>&1 是 stderr to stdout，即 stdout 輸出到哪裡，stderr 就跟著到那裡，
# 以這裡為例就是將 stdout 及 stderr 都輸出至 file4 的意思。

barry@suse:~> cat /etc/passwd noexistfile > file5 1>&2
# 1>&2 是 stdout to stderr，即 stderr 輸出到哪裡，stdout 就輸出到那裡，
# 所以這時候 stdout 及 stderr 都會輸出至螢幕。

barry@suse:~> cat /etc/fstab noexistfile > /dev/null 2>&1
# 不想讓資料輸出至螢幕或檔案時，就可將其轉向輸出至 /dev/null 這個無底洞的特殊裝置裡。

# 以下介紹幾種清空檔案內容的方式：
barry@suse:~> true > file1
# 執行 true 指令時，不會輸出任何訊息，且代表著這個指令的執行是成功的。
barry@suse:~> cat /dev/null > file2
barry@suse:~> cat > file3
# 最後一種方式是不做任何輸入而直接按下 Ctrl-D 來作結束。 


指令 >> 檔案

　這個重導符號也是轉向輸出的一種。當檔案不存在時，會先建立起這個檔案，再把 stdout 的資料放進來 ﹔當檔案存在時，則會將 stdout 的資料附加至檔案內容的後面，而不會覆蓋原來檔案的內容。

範例說明：
barry@suse:~> cat /etc/passwd > barryfile
barry@suse:~> cat /etc/fstab >> barryfile
# 將 /etc/fstab 的內容附加至 barryfile 內。

指令 < 檔案

　"<" 是轉向輸入的重導符號，最常見的是以檔案內容來代替原本的鍵盤輸入。

範例說明：
barry@suse:~> cat < /etc/fstab
# 這是以 fstab 的檔案內容代替原本從鍵盤所做的 stdin，並交給 cat 指令來讀入，最後當然
# 就是將所讀入的資料列印在螢幕上囉。
# 執行 "cat < /etc/fstab" 就等於執行 "cat 0< /etc/fstab"，因 FD 0 為預設。
barry@suse:~> mail tina < messagefile
# 使用 mail 指令來傳訊息給其他人時，mail 預設會從 stdin 取得您所輸入的訊息，而在
# 使用了轉向輸入的重導符號後，即能以檔案內容來替代原本的鍵盤輸入。

barry@suse:~> cat < file1 > file2
# 以 file1 的內容當成 stdin，並由 cat 讀入後轉向輸出至 file2。

指令 << 字串

　這也是轉向輸入的重導符號，不過只有當您所輸入的字串，符合 "<<" 右邊的字串內容時，才代表輸入的結束。

範例說明：

barry@suse:~> cat << endoff
> barry　　← 輸入 barry。
> mary 　　 ← 輸入 mary。
> endoff　 ← 輸入 endoff。
barry
mary
# 表示從 keyboard 做輸入時，一直要輸入至 endoff 字串時才會結束，這時候就會將
# 剛剛所做的輸入內容輸出到螢幕。

barry@suse:~> cat << end > barryfile
# 輸入至 end 字串後結束，並且將之前所輸入的內容輸出到 barryfile。



3.3.3 管線 pipe ( | )

　管線的主要用途，在於將管線前面指令的標準輸出當成管線後面指令的標準輸入，像 cmd1 | cmd2 | cmd3 就是將 cmd1 的標準輸出當成 cmd2 的標準輸入，接著再將 cmd2 的標準輸出當成 cmd3 的標準輸入。

　另外要注意一點，管線預設只能接受前面指令的標準輸出 (stdout)，不包括標準錯誤輸出 (stderr)，除非於指令後方使用 "2>&1"，這樣才能把 stdout 及 stderr 都交給管線之後的指令來處理。

範例說明：
barry@suse:~> cat /etc/termcap | less
# 把 cat 指令的標準輸出(stdout)當成 less 指令的標準輸入(stdin)，也就是說 less 指令的
# 標準輸入會接受來自 cat 指令的標準輸出，因此 cat 指令的執行結果就可以被 less 所讀入，
# 然後即可讓我們使用 [Page Up]、[Page Down] 來翻頁查看內容囉。

barry@suse:~> cat /etc/fstab | cat -n
# 把第一個 cat 指令的標準輸出當成第二個 cat 指令的標準輸入，所以就能
# 把 /etc/fstab 的內容附上編號後 print 出來囉 !

barry@suse:~> cat /etc/fstab nofile | nl > file
cat: nofile: No such file or directory
# 由於管線之後的 nl 指令，只能接受管線前面 cat 指令的 stdout，所以 file 檔案內容裡，
# 不會包括 stderr 的訊息。如果要把 stdout 及 stderr 都能輸出至 file，則可執行如下：
 barry@suse:~> cat /etc/fstab nofile 2>&1 | nl > file

　以下介紹兩個比較特殊的指令來搭配管線，看看如何使用。

指令 tee

　tee 指令只能從 stdin 讀取，且會把結果同時輸出至檔案及螢幕上。

範例說明：
barry@suse:~> nl /etc/passwd | tee /tmp/testfile | tac
# 將檔案內容每一行附上編號後交由 tee 指令處理，此時 tee 就會把結果同時輸出至
# 螢幕及 /tmp/testfile 裡，最後再利用 tac 指令來把輸出至螢幕的內容反向做排序。

# 另外如果tee 之後是接一個已經存在的檔案，則可加上 -a 參數來避免覆蓋原來檔案的內容：
barry@suse:~> cat /etc/fstab | tee -a /tmp/testfile

指令 xargs

　可把前面指令的執行結果當成 xargs 後面所接指令的引數。

範例說明：
barry@suse:~> ls | xargs rm
# rm 是用來刪除檔案的指令。
# 此例是說將 ls 的執行結果當成 rm 指令的引數，所以家目錄下的檔案就會被刪除掉。


3.4 過濾檔案內容的指令

　這節要介紹一些 filter commands，也就是說平常我們可以利用這些指令來取出檔案的部分內容，或者在調整檔案內容後，將修改過後的結果做標準輸出或轉向輸出。

　在 Linux 上的過濾指令有很多，以下只會列舉幾個基本的過濾指令來介紹。

指令 head

　用來輸出檔案內容前面幾行的指令。當未指定輸出的行數時，則預設是列印前十行。

指令語法：
head [OPTION]... [FILE]...

參數說明：
-n 指定要輸出檔案前幾行的內容。

範例說明：
barry@suse:~> head -n 3 /etc/passwd
root:x:0:0:root:/root:/bin/bash
bin:x:1:1:bin:/bin:/bin/false
daemon:x:2:2:daemon:/sbin:/bin/false
# 列出檔案前 3 行的內容。也等於以下兩式：
barry@suse:~> head -3 /etc/passwd
barry@suse:~> cat /etc/passwd | head -3

barry@suse:~> nl /etc/passwd | head -5
# 將檔案內容附上編號後，取前面五行來做輸出。

指令 tail

　用來輸出檔案內容後面幾行的指令。當未指定輸出的行數時，則預設是列印最後面十行。

指令語法：
tail [OPTION]... [FILE]...

參數說明：
-n 指定要輸出檔案後幾行的內容。
-f 當搭配這個參數時，可用來監視檔案內容中的訊息變動。

範例說明：
barry@suse:~> tail -n 3 /etc/passwd
barry@suse:~> tail -3 /etc/passwd
barry:x:1000:100::/home/barry:/bin/bash
mary:x:1001:100::/home/mary:/bin/bash
tina:x:1002:1000::/home/tina:/bin/bash
# 列出檔案最後三行的內容。
barry@suse:~> tail +20 /etc/passwd
# 將檔案內容從第 20 行至最後一行列印出來。
suse:~ # tail -f /var/log/messages
Jan 9 03:20:10 suse pppd[2930]: Script /etc/ppp/ip-up finished (pid 3234), status = 0x0
Jan 9 03:20:11 suse kernel: eth0: no IPv6 routers present
Jan 9 03:20:12 suse zmd: NetworkManagerModule (WARN): Failed to connect to NetworkManager
Jan 9 03:20:14 suse [acpid]: client connected from 3446[0:0]
Jan 9 03:20:14 suse [acpid]: 1 client rule loaded
Jan 9 03:20:16 suse ip-up: postqueue: fatal: Cannot flush mail queue - mail system is down
Jan 9 03:20:28 suse zmd: Daemon (WARN): Not starting remote web server
Jan 9 03:22:50 suse sshd[3573]: Accepted password for root from 192.168.1.226 port 2220
Jan 9 03:39:55 suse -- MARK --
Jan 9 03:52:33 suse sshd[3733]: Accepted password for root from 192.168.1.226 port 2479
# 瀏覽 /var/log/messages 紀錄檔內最後十行的訊息內容。此時您會注意到 tail 程式還是在
# 執行狀態，並沒有結束，如果此時又有最新訊息進來，您便可以在螢幕上看得一清二楚。
# 要結束 tail 的執行，請按下 Ctrl- C。
suse:~ # tail -f /var/log/messages >> logfile &
# 最後面那個 "&" 是背景執行符號，這樣一來，程式就不會佔用前景終端機來執行，
# 而您只要隨時檢查 logfile， 就可以知道最新的紀錄訊息。
# 將 /etc/passwd 檔案內容附上編號，並列出第15至20行：
 barry@suse:~> nl /etc/passwd | head -20 | tail -6
　15 named:x:44:44:Name server daemon:/var/lib/named:/bin/false
　16 postfix:x:51:51:Postfix Daemon:/var/spool/postfix:/bin/false
　17 messagebus:x:100:101:User for D-BUS:/var/run/dbus:/bin/false
　18 sshd:x:71:65:SSH daemon:/var/lib/sshd:/bin/false
　19 haldaemon:x:101:102:User for haldaemon:/var/run/hal:/bin/false
　20 mysql:x:60:103:MySQL database admin:/var/lib/mysql:/bin/bash
# 管線之後先使用 head 指令來取出前20行，然後再利用 tail 指令取出這20行的後六行。

指令 cut

　此指令可用來擷取檔案中每一行的部分資料。

指令語法：
cut [OPTION]... [FILE]...


參數說明：
-c 只擷取字元。
-f 只擷取欄位，而每個欄位預設是以 Tab 鍵來作為欄位之間的區隔。
-d 如不使用預設的 Tab 鍵來區分欄位，即可使用這個參數來額外指定。

範例說明：
barry@suse:~> cat /etc/passwd | cut -c 2,5
# 列出檔案中每一行的第 2 及第 5 個字元。

 barry@suse:~> cut -c 2-5 /etc/passwd
# 列出檔案中每一行的第 2 至第 5 個字元。

barry@suse:~> cut -d '：' -f 2,4 /etc/passwd
# 以 ":" 做欄位之間的區隔，而取出每一行的第二及第四欄位。

barry@suse:~> cut -d ' ' -f 3-5 file
# 欄位之間是以空白字元做區隔，來列出第三至第五欄位。
# 請留意兩個單引號之間存在著一個空白字元噢。

指令 wc

　可利用這個指令來計算檔案內容總共有幾行、幾個字、幾個字元。

指令語法：
wc [OPTION]... [FILE]...
　
參數說明：
-l 同於長參數 --lines。計算檔案內容總共有幾行。
-w 同於長參數 --words。計算檔案內容總共有幾個字。
-c 同於長參數 --bytes。計算檔案內容總共有多少 bytes，就是多少字元的意思。

範例說明：
barry@suse:~> wc /etc/passwd /etc/fstab
　30　67 1582 /etc/passwd
　12　72　948 /etc/fstab
　42 139 2530 total
# 個別顯示兩個檔案有幾行、幾個字及幾個字元，並統計總數。
barry@suse:~> cat /etc/passwd | wc -l
30
# 顯示 passwd 檔案共有 30 行。/etc/passwd 是個帳號密碼檔，每一行是代表一個帳號資訊，
# 而這裡顯示30行，就表示系統上總共有30個帳號存在。
barry@suse:~> cat /etc/passwd | wc -wc
　67 1582 /etc/passwd
# 顯示 passwd 檔案共有 67 個字及 1582 個字元。

指令 tr

　若要執行字元的替代 (轉換) 或刪除，可使用這個指令。

指令語法：
tr [OPTION]... SET1 [SET2]

　SET1 為被替代或刪除的字元組合。如以替代功能而言，則 SET2 就代表是要替代 SET1 的字元組合，比如 "tr ab 12" 的意思就是將 a 字元以 1 來替換，b 字元以 2 來替換，且是一個對一個的替代。另外 tr 指令只能從 stdin 來讀取資料。

參數說明：
-d 將檔案內容中，含有此參數所指定的字元刪除。使用此參數時，不能執行替代用法。
-s 當您想把檔案內容中，所出現連續重複的字元刪除至剩一個時，就可以使用這個參數。

範例說明：
barry@suse:~> cat /etc/passwd | tr bary abcd
# 將檔案中的 b 字元換成 a，a 換成 b，r 換成 c，y 換成 d，執行完成後會將結果輸出到螢幕。
# tr 這個指令必須從 stdin 讀取資料，所以一般會搭配 "|" 或 "<" 來使用。
 
barry@suse:~> cat /etc/passwd | tr [a-z] [A-Z]
# 將檔案中的所有小寫英文字母替換成大寫英文字母，並將最後結果輸出到螢幕。
barry@suse:~> cat file | tr -s ro
# 若檔案內容中的 r 或 o 字元有連續重覆的，將其縮減成為一個。
barry@suse:~> cat file1 file2 | tr -d cX
# 刪除檔案中的 c 及 X 字元，並將執行後的內容輸出到螢幕。

指令 sed

　sed 指令是個資料流編輯器，可針對資料進行新增、替代、刪除、列印指定行數等等的動作，並把處理過後的資料做標準輸出。

指令語法：
sed [OPTION]... 'Action' [input-file]...

　input-file 是指定 sed 要處理哪些檔案裡的資料內容，當然您亦可把 sed 置於管線之後，這樣就可以針對前面指令的執行結果進行進一步的處理。

　而 Action 即是對於這些資料內容所即將要採取的處理動作，像是要做字串替代或刪除等行為。

參數說明：
-e 於命令列上直接編輯 Action。一般不加此參數也行。
-f 事先將 sed 的 Action 寫在檔案內，然後在命令列上使用 -f file 就能執行這個 Action。
-n 只針對有被處理過的那幾行內容做輸出。預設是全部做輸出的。
-r 使 sed 能夠支援延伸式的正規表示法 (參考 3-5 節)。

Action 中的編輯指令：
指 令 代　表　意　義
a 這是新增資料用法，比如 '3a i am barry' 就是在第三行之後 (第四行) 新增 i am barry 這一行。
i 這是新增資料用法，比如 '3i i am barry' 就是在第三行之前 (第二行) 新增 i am barry 這一行。
d 這是刪除用法，比如 '5,8d' 就是刪除第 5~8 行。
p 列印之意囉，通常會搭配 -n 參數一起使用。
s 這是替代用法，常搭配 p (print) 及 g (global) 參數來使用。比如 '3,8s/bash/false/g' 就是把第 3~8 行中的 bash 字串全替代成 false。

範例說明：

barry@suse:~> nl /etc/passwd > file
# 先製造一個檔案 file，並將內容附上編號，待會兒就以 file 來做範例。

barry@suse:~> sed '3a I am barry.\
> Hello,everybody.' file
# 於第三行之後 (即第四行開始) 附加 I am barry. 及 Hello,everybody. 這兩句。
# 當新增多行時，於每一行末端加上 \ 即可。

barry@suse:~> sed '20d' file
# 刪除檔案第 20 行，並將處理後的結果輸出至螢幕。
# 如要將改變後的結果儲存起來，可用 ">" 來輸出至另一個檔案。

barry@suse:~> sed '10,20d' file
# 刪除檔案的第 10 至 20 行。

barry@suse:~> sed '5,$d' file
# 將檔案內容從第五行刪除至最後一行。

barry@suse:~> sed '/false/d' file
# 將檔案中含有 "false" 的那幾行全部刪除。

barry@suse:~> sed 's/barry/michael chen/' file
　25 michael chen:x:1000:100::/home/barry:/bin/bash
# 檔案中含有 barry 字串的，就替代成 michael chen。或許各位已注意到只替代了第一個
# barry，如想把所有的 barry 全替代成 michael chen，只需於結尾加上 g 參數即可：
barry@suse:~> sed 's/barry/michael chen/g' file
　25 michael chen:x:1000:100::/home/michael chen:/bin/bash

# 另外您也可以指定要替代第幾個字串，比如：
barry@suse:~> sed 's/barry/michael chen/2' file
　25 barry:x:1000:100::/home/michael chen:/bin/bash
# 這樣第二個 barry 就會被替代成 michael chen 了。

barry@suse:~> sed '15,$s/bash/false/g' file
# 將 bash 替代成 false，不過只針對第 15 行到最後一行。

barry@suse:~> sed 's/barry/michael/2;s/mary/vivian/1' file
# 這是組合式的用法，表示將第二個 barry 替代成 michael，第一個 mary 替代成 vivian。

barry@suse:~> sed -n '16,18s/false/true/p' file
　16 named:x:44:44:Name server daemon:/var/lib/named:/bin/true
　17 vscan:x:65:103:Vscan account:/var/spool/amavis:/bin/true
　18 dhcpd:x:102:65534:DHCP server daemon:/var/lib/dhcp:/bin/true
# 這是 -n 搭配 p 的範例，如此就只列印出有改變的那幾行。

barry@suse:~> sed -nf editfile file
# 事先將 Action 寫入 editfile 裡，然後用 -f 來指定此 editfile。
# 您可做個實驗，先把 16,18s/false/true/p 放入 editfile 中，接著在命令列上執行看看。

barry@suse:~> sed -n '5,8p' file
　5 mail:x:8:12:Mailer daemon:/var/spool/clientmqueue:/bin/false
　6 games:x:12:100:Games account:/var/games:/bin/bash
　7 wwwrun:x:30:8:WWW daemon apache:/var/lib/wwwrun:/bin/false
　8 ftp:x:40:49:FTP account:/srv/ftp:/bin/bash
# 只列印出第五行至第八行。



3.5 正規表示法 (RE) 及萬用字元 (Wildcard)

　RE (Regular Expressions：regexp) 就是所謂的正規表示法，他是經由我們所輸入的一般字元或特殊字元的組合，其主要用途是用來對所指定的檔案內容，或經由指令輸出到螢幕的資訊去做搜尋、刪除或替代等功能，當然您還要搭配支援 RE 的程式來使用才行。

　舉個例子，比如現在有一個內容很大的檔案，可能有好幾千行，若此時想找尋特定關鍵字的那幾行，可能找個老半天還不一定找得出來，這時候就可以使用一些支援 RE 的工具再搭配 RE 來幫我們做搜尋的工作，這樣您就可以不費吹灰之力的找到想要的結果。

　在搭配這些正規表示法的工具當中，最常使用的就是 grep、sed 及 awk 指令，而這裡主要是針對 grep 來做說明，至於 sed 部分稍後也會做一點補充。

　另外在 shell 下也常會搭配一些特殊字元來方便操作，比如 "rm test*"，代表的意思是刪除檔名中以 test 做開始的所有檔案，而那個 "*" 就是最常用的萬用字元。至於其他的詳細說明，請參考以下各小節。　

3.5.1 正規表示法

　學習 RE 之前，可先要好好了解一下 grep 這個指令的用法，因為在搭配 RE 的工具當中，grep 應該是使用頻率極高的一個指令。因此這個地方就先來學學 grep 指令的用法，接著當然就會搭配著 RE 來一起使用了。

指令 grep

　grep 主要功用是把我們所指定的字串樣式，拿去跟資料內容做比對，然後把內容中含有該字串的那幾行列印出來。再說白話一點，就是搜尋檔案內容或輸出至螢幕的資料中，含有 grep 其後所指定字串樣式 (pattern) 的那幾行找出來啦。

指令語法：
grep [options] 'PATTERN' [FILE...]

參數說明：
-c 只統計合乎搜尋條件的總共有幾行。
-E 加上這個參數才可以支援延伸式的正規表示法。grep -E 就相當於 egrep 這個單一指令。
-i 忽略 pattern 中的大小寫。
-n 將符合搜尋條件的那幾行，附上編號列印出來。
-q 加上這個參數後，可防止任何搜尋到的資料做 stdout 。
-r 遞迴搜尋所指定的目錄中，檔案內容含有該 pattern 的那幾行。。
-v
把不符合搜尋條件的那幾行列印出來。
-w
找出 pattern 是一個 word 的那幾行。

基本範例：
barry@suse:~> cat /etc/passwd | grep 'bash'
barry@suse:~> grep 'bash' /etc/passwd
# 夾在單引號中的 bash，就是要與 /etc/passwd 的檔案內容做比對的 pattern，
# 只要包含 bash 這個 pattern 的那幾行都會被搜尋出來。

　RE 裡頭也可以包含一些特殊字元，只不過在剛剛那個範例裡頭都是使用一般字元罷了，所以緊接著就來看看 RE 裡頭特殊字元所代表的意義：

RE 特殊字元
代　表　意　義
.
 表示任一單一字元，包括空白字元在內。
[abc]
 表示含有 [ ] 內的單一字元，若寫成 [a-z,0-9]，就代表含有小寫英文字母或數字的單一字元。
[^abc]
 表示非 [ ] 內的單一字元，也就是不含有 abc 的任一單一字元。
*
 在 "*" 的前面一個字元可以出現 0 ~ n 次。
^abc
 表示該行是以 abc 為首。
abc$
 表示該行是以 abc 為尾。
\
 這是跳脫符號，用來跳脫置於其後的特殊字元，使其變成一般字元。
\<word\>
 將檔案內容中，含有 word 這個字的那幾行找出。word 的意思是說該 pattern 左右兩邊無字母或數字者，即可稱為一個 word，比如 barry# 中 barry 就是一個 word，又如 3barry 中的 barry 就不是一個 word。
\{n\}
 表示前面一個字元剛好出現 n 次。
\{n,\} 表示前面一個字元至少出現 n 次。
\{n,m\}
 表示前面一個字元可以出現 n 到 m 次。
? 在 "?" 的前面一個字元可以出現 0 ~ 1次。需搭配 egrep (extended RE) 或 grep -E 指令來使用。
+ 在 "+" 的前面一個字元可以出現 1~ n 次。需搭配 egrep 或 grep -E 指令來使用。
str1|str2 搜尋包含 str1 或 str2 的那幾行。需搭配 egrep 或 grep -E 指令來使用。

　在以上表格中的最後三個字符，即是延伸式正規表示法中的特殊字元，請記得搭配 grep -E 或 egrep 指令來使用。

　以下舉個範例來說明。假使在 barry 家目錄下有個檔案叫 file ，其內容如下：

barry@suse:~> cat file
barry marytina
xybrry 123xyz

123aba*rry
user01 wcky
auser02 test good
buser03 nice priority
cuser04 lpi
abuser05 www lpi org
xyzuser06 study area
abc 123!
123abcxyz
ab3 cwy

　接著就利用這個檔案做範例，來驗證剛剛所提及的 RE 觀念。

barry@suse:~> cat file | grep 'ab..c'
ab3 cwy
# 列出在 ab 與 c 之間夾兩個任一單一字元的這個 pattern 的那幾行。

barry@suse:~> cat file | grep '[abc]user'
auser02 test good
buser03 nice priority
cuser04 lpi
abuser05 www lpi org
# 找出檔案中含有 auser、buser 或 cuser 的那幾行。

barry@suse:~> grep 'x.z' file
xybrry 123xyz
xyzuser06 study area
123abcxyz
# 列出 xz 之中夾個任意單一字元的這個 pattern 的那幾行。

barry@suse:~> cat file | grep '[^bc]user'
auser02 test good
xyzuser06 study area
# 在 user 的前面一個字元，只要不是 b 或 c ，而是其他字元的話，皆符合要搜尋的 pattern。

barry@suse:~> cat file | grep 'ba*rry'
barry marytina
xybrry 123xyz
# a 字元可以出現 0 ~ n 次，所以只要包含 brry、barry、baarry、… 等等這些 pattern 的
# 那幾行都能被找出。

barry@suse:~> cat file | grep '^user'
user01 wcky
# 檔案內容裡，只要該行是以 user 為首的，都會被列出來。

barry@suse:~> cat file | grep 'xyz$'
xybrry 123xyz
123abcxyz
# 只要該行是以 xyz 為尾的，都能被搜尋出來。

barry@suse:~> cat file | egrep 'ba?rry'
barry marytina
xybrry 123xyz
# a 字元可以出現 0 或 1 次，所以只要包含 brry 或 barry 的那幾行就能被搜尋出來。

barry@suse:~> cat file | egrep 'ba+rry'
barry marytina
# pattern 中的 a 字元可以出現 1到 n 次。

barry@suse:~> cat file | grep 'ba\*rry'
123aba*rry
# 由於在 "*" 之前加上個跳脫符號 "\" ，所以 "*" 會被當成一般字元來處理，
# 而不再具有 RE 裡的特殊涵義。

barry@suse:~> grep '\<[0-9]\{3\}\>' file
abc 123!
# 夾在 " \< " 及 " \> " 之間的是一個 word，而這個 word 是由三個數字所組成的，
# 所以只要 pattern 是由三個數字所組成的 word，就符合搜尋的條件。

barry@suse:~> egrep 'test|nice' file
auser02 test good
buser03 nice priority
# 列出含有 test 或 nice 這個 pattern 的那幾行。

barry@suse:~> grep '^$' file

# 這是個比較特殊的用法，是用來搜尋空白列的。

barry@suse:~> egrep '^[A-Za-z0-9]+$' file
123abcxyz
# 上個範例的 pattern 為"^$" ，是表示搜尋空白行，那如果 pattern 為 "^abc$"，就表示
# 要搜尋的該行內容就為 abc。所以這個例子是在搜尋檔案內容中只包含字母及數字
# 的那幾行，且該行不能含有空白字元。

Tips：當命令列上的特殊字符被單引號包住時，大部分都無法被 shell 所解析，而 grep 之後的 pattern 之所以放在單引號裡，就是為了避免提早被 shell 解析掉，如此方能將 RE 裡的特殊字符完整的交給支援 RE 的指令來處理。

　基本的 grep 操作沒問題後，接著就來看看加上參數後的範例，而這個地方還是沿用上面那個 file 的內容。

barry@suse:~> cat file | grep -c 'user'
6
# 只統計符合搜尋條件的總共有幾行。
barry@suse:~> cat file | grep -vn 'user'
1:barry marytina
2:xybrry 123xyz
3:
4:123aba*rry
11:abc 123!
12:123abcxyz
13:ab3 cwy
# 將不符合搜尋條件的那幾行，附上編號列出來。

barry@suse:~> grep -w '123' file
abc 123!
# 將檔案中含有 123 這個字的那幾行找出來。
barry@suse:~> grep -i 'BARRY' file
barry marytina
# 在忽略大小寫的前提下，搜尋包含 barry 這個 pattern 的那幾行。
suse:~ # grep -r 'pure-ftpd' /usr/share/doc
# 遞迴搜尋 /usr/share/doc 目錄裡邊的檔案內容，包含 pure-ftpd 這個 pattern 的那幾行。

sed 指令搭配 RE 的範例

　對於 sed 指令的基本用法應該還記得吧，而這裡主要是補充 sed 搭配 RE 的使用方式。一般常見到 RE 出現在 sed 裡 Action 中的位置如下：

　˙ s/regexp/replacement/
　˙ /regexp/

　至於詳細的使用方式請參考以下的範例：

barry@suse:~> sed -n '/^[a-z]user/p' file
auser02 test good
buser03 nice priority
cuser04 lpi
# user 字串之前是任何一個小寫字母，且該行是以此為首的話，就把這幾行列印出來。

barry@suse:~> sed '/\<false\>/d' /etc/passwd
# 將檔案內容中含有 false 這個 word 的那幾行刪除，並將結果輸出到螢幕。

barry@suse:~> sed -n 's/ba*rry/111/p' file
111 marytina
xy111 123xyz
# 只要 pattern 是 brry、barry、baarry、…，就以 "111" 將其替代掉，
# 並且只輸出有替代的那幾行。

barry@suse:~> sed '/^$/d' file > file1
# 刪除空白列後，將結果輸出到 file1。

barry@suse:~> sed -nr 's/ba?rry/michael/gp' file
michael marytina
xymichael 123xyz
# 由於 RE 裡頭有包含 "?" 字符，因此要加上 -r 參數，如此方可支援延伸式的 RE。


3.5.2 萬用字元 (Wildcard)

　為了方便在命令列上的操作，往往可以運用一些萬用字元符號於其中。以下列舉幾個 Wildcard 給各位參考：
Wildcard
代　表　意　義
*
 表示 0 ~ n 個字元。
?
 表示任意單一字元，包括空白字元。
\ 跳脫特殊字元的符號，使其變成一般字元。
[abc]
 表示 [ ] 內的單一字元。
{ab,cd}
 將 { }內的字串展開。
[!a-z]
 ! 是 not 的意思，即除了小寫字母外的任意單一字元。

範例說明：

barry@suse:/tmp> ls
afile　fi le　file　file name　file5　fileC　fileD　filea　fileab　fileb
# 這是目前 /tmp 目錄下所存在的檔案。請留意 "fi le" 及 "file name" 兩檔名間有空白字元噢。

barry@suse:/tmp> ls file*
file　file name　file5　fileC　fileD　filea　fileab　fileb
# 列出檔名是以 file 做開始，其後可接 0 ~ n 個字元的檔案。
# 假使這個目錄下還存在一個叫 filedir 的目錄，則 filedir 的內容也會列印出來噢。

barry@suse:/tmp> ls ?????
afile　fi le　file5　fileC　fileD　filea　fileb
# 列出檔名為五個字元的檔案。

barry@suse:/tmp> ls file[a-z]
filea　fileb
# 列出檔名是以 file 做開始，名稱後接的是一個小寫的英文字母。
# 如您在實做時不是出現這樣的結果，可先於命令列上執行：
# "export LC_ALL=C"，然後再試試。

barry@suse:/tmp> ls file{a,b}
filea　fileb
# 將 {a,b} 內的字串展開，所以就會列出 filea 及 fileb 了。

barry@suse:/tmp> ls file[!a-e]
file5　fileC　fileD
# 列出 file 名稱後還接一個非 a ~ e 的單一字元。

barry@suse:/tmp> cat file*
# 瀏覽檔名是以 file 為首的檔案內容。

barry@suse:/tmp> cat file\*
# 這表示要瀏覽的是 file* 這個檔案。

barry@suse:/tmp> touch {ab,cd}{e,f}
barry@suse:/tmp> ls
abe abf cde cdf
# touch 指令可用來建立空檔案，而利用這種方式，可以讓我們一次建立好幾個空檔案。

barry@suse:/tmp> rm file*
# 刪除所有以 file 名稱做開始的檔案。



3.5.3 在命令列上使用 \ 、 ; 、&& 、 || 及 {指令群組}

　從 3-2 節至 3-5 節，我們介紹了一些基本指令的操作，以及輸入、輸出的觀念，而往往在命令列上您會將它們一起搭配來使用，此時所輸入的指令敘述可能會超過一行，因此在操作習慣上，可以在第一行的句末加上個跳脫符號 ( \ )，然後再按下 [Enter] 鍵以後就可以繼續做輸入。

　平常在命令列上輸入指令後會敲下 [Enter] 鍵，這個動作其實就是送出一個 CR (Carriage Return) 字符來給 shell，shell 只要接收到這個 CR 字符，就會開始解析您所輸入的指令。如果在您敲下 [Enter] 鍵以前先輸入 "\"，這樣就可以把 CR 字符給跳脫出去，因此可以讓您繼續輸入未完成的指令敘述。

範例：
barry@suse:~> cat /etc/termcap | head -200 | tail +100 | cat -n \
> | sed 's/terminal/TERMINAL/' > /tmp/barryfile

　另外想要在命令列上個別執行多個指令，則可用 ";" 做指令之間的區隔，且不論前面指令執行成功與否，都會繼續執行下一個指令。

範例：
barry@suse:~> date ; nocmd ; pwd
Wed Jan 10 17:59:30 CST 2007
-bash: nocmd: command not found
/home/barry

　再來就是 && 及 || 的介紹。使用 && (and) 來連接兩個或多個指令時，表示第一個指令在成功被執行後，下一個指令才能夠繼續執行，那若第一個指令執行失敗，則下一個指令將不會被執行 ﹔而 || (or) 則表示第一個指令執行失敗後才會繼續執行下一個指令，如第一個指令成功被執行，則下一個個令將不會被執行。

　所以簡單的說，"cmd1 && cmd2" 表示只有在 cmd1 執行成功後才會執行 cmd2 ﹔"cmd1 || cmd2" 表示只有在 cmd1 執行失敗後才會執行 cmd2。

範例：
barry@suse:~> date && pwd && cd /tmp
Wed Jan 10 18:07:21 CST 2007
/home/barry
barry@suse:/tmp>
# 因 date 指令能成功執行，所以會接著執行 pwd，又因 pwd 也能正確執行，
# 所以最後會執行 cd /tmp。

barry@suse:~> nocmd || date && pwd
-bash: nocmd: command not found
Wed Jan 10 18:12:46 CST 2007
/home/barry
# 因 nocmd 無法成功執行，所以會接著執行 date，而當 date 能被正確執行後，
# 才會繼續執行 pwd。

　最後還有個指令群組的介紹。當要在命令列上輸入多個指令，並將所有指令的執行結果利用管線或重導符號來處理時，就可以把這些指令放在 { } 中，看以下範例就知道了：

barry@suse:~> { date;pwd;cat /etc/passwd; } > /tmp/file
# 請留意在 "{" 與指令之間需留有空格。

3.6.3 command-line mode (extended mode)

　從 command mode 中可以輸入「:」冒號來進入到 command-line mode，而在冒號後面就可以輸入一些指令來操作，請參考以下的範例：

EX1：
：s/str1/str2
將 str1字串替代成 str2。請注意，若剛剛在命令模式下，游標位置不是停留在含有 str1的那一行，這時候執行替代時會出現 "Pattern not found" 的錯誤訊息。另外替代指令的結尾處有無加上 "/" 都可以。

EX2：
：20,$s/str1/str2/g
從第二十行至最後一行進行全域替代。

EX3：
：5,$d
從第五行刪除至最後一行。

EX4：
：$
將游標移至最後一行。

EX5：
：100
將游標移到第一百行。

EX6：
：wq
存檔離開 (write and quit)。輸入 x 如「：x」，亦可達相同的效果。

EX7：
：w
寫入存檔的意思，不過尚未離開 vi，您可以在編輯檔案過程的任何時候執行這個指令。另外有時侯您可能對某個檔案只具備唯讀的權限，所以在修改此檔後要執行 w 時會失敗，此時如果您是這個檔案擁有者，那可以執行「:w!」來強迫寫入噢。

EX8：
：q!
強迫離開未存檔。這樣做可以取消之前對檔案內容所做的任何更動。

EX9：
：wq!
強迫寫入並存檔離開。

EX10：
：w file1
將目前檔案內容寫入另外一個檔案 file1 中。

EX11：
：e file2
開啟另外一個檔案 file2 來編輯。

EX12：
：r file3
將 file3 檔案內容附加至原本檔案內容中游標所在位置的下面。

EX13：
：set nu
將檔案內容附上編號。

EX14：
：set nonu
取消所設定的編號。

EX15：
：syntax on
如果想讓檔案內容依照不同的設定語法顯示不同的顏色，那就執行這個指令。您可以練習使用 vi 開啟 /etc/fstab，然後在 command-line mode 底下執行這個指令試看看。

EX16：
：syntax off
關閉這個功能。

EX17：
：!ls -ld /var
command-line mode 中，可於 ! 之後加上您要執行的指令，像這個地方就是在查看 /var 目錄的長格式。

3.6.4 暫存檔說明
　
　當使用 vi 在編輯檔案時，會同時產生一個暫存檔在此目錄下，比如在編輯 ~/testfile 時，就會產生一個 .testfile.swp 檔案在您家目錄中，此時您所做的任何修改，都會暫時存放在這個檔案裡面，而就在編輯完畢正常存檔離開以後，這個暫存檔也會跟著消失，那萬一因為某一些原因造成 vi 程式執行的中斷，那麼這個暫存檔就會被保留在家目錄之中，而當下次重新使用 vi 開啟這個檔案時，於螢幕最下方就會出現這樣的訊息：

Swap file ".testfile.swp" already exists!
[O]pen Read-Only,　(E)dit anyway,　(R)ecover,　(Q)uit,　(A)bort,　(D)elete it:

　這時候可以按下 Q 來選擇離開 vi，或輸入 D 來刪除這個 Swap file，不然也可以輸入 R 來回復當初修改的內容至原始檔案裡，不過等回復完成存檔離開後，可記得要將這個暫存檔刪除，不然下次又開啟這個檔案時，還是會看到同樣的畫面。

　另外如果之前都無任何程式中斷的紀錄，但卻還是看到這個畫面，那是因為重複開啟檔案的結果，您可以嘗試於 tty1 使用 vi 去開啟一個檔案，再到 tty2 去開啟同一個檔案試看看，只不過這次所看到的畫面中沒有 (D)elete 這個項目而已。

3.7.1 echo 及 alias 指令　

　對於等一下要介紹的變數而言，echo 指令可是很重要的呢，因為要藉由這個指令，來讓變數的內容能呈現在我們面前。

　 至於另一個 alias 指令，則是用來設定指令別名用的，至於先介紹這個指令的原因，是因為在環境設定檔那一節裡會用到這個指令來做範例，所以囉，就先介紹一下囉。

指令 echo

　此指令一般用來對其後所輸入的字串做標準輸出。　

指令語法：
echo [OPTION]... [STRING]...

參數說明：
-n 取消句末的換行字符。
-e 啟用反斜線控制字符。

　當執行 echo 指令時，預設會自動幫您於句末補上一個換行字符，所以在輸出 STRING 後就會換行至下一行，但加上 -n 參數時就會取消此換行字符，參考以下範例就會有概念：
barry@suse:~> echo good
good

barry@suse:~> echo -n good
goodbarry@suse:~>

　至於反斜線控制字符常見的有：

　　˙\a：發出警告嗶嗶聲 (alert)。
　　˙\b：相當於您按下 [Back Space] 鍵。
　　˙\c：取消句末的換行字符 (cancel)。
　　˙\n：換行字符 (newline)。
　　˙\t：相當於您按下 [Tab] 鍵。

範例說明：

barry@suse:~> echo good boy
good boy
# 這是最基本用法，就是把 echo 後的字串做標準輸出而已。

barry@suse:~> echo good 　　boy
good boy
# 是否有發現到輸出的內容並沒有保持原來的間距，如欲維持原間距，
# 就使用引號將其包住：
barry@suse:~> echo "good 　　boy"
good 　　boy

barry@suse:~> echo `date`
Thu Jan 11 02:25:07 CST 2007
# date 是一支執行程式，所以需用反引號 (backquote) 將 date 包起來，這樣 shell 才
# 知道它是一支程式而不是一般字串，因而能夠被 shell 所優先處理 。

# 您亦可使用以下的方式來達到與上式同樣的目的：
barry@suse:~> echo $(date)
Thu Jan 11 02:28:34 CST 2007

barry@suse:~> echo '`date`'
`date`
# echo 後面如使用單引號，則 shell 會將反引號這個特殊字符當成一般字元處理。

barry@suse:~> echo current time is `date`
current time is Thu Jan 11 02:30:46 CST 2007
# 這應該沒啥問題吧。

barry@suse:~> echo my name is \"barry\"
my name is "barry"
# 利用 \ 這個跳脫符號，就能將雙引號顯示出來。

barry@suse:~> echo -e "my\tname\nis\t\bbarry\c"
my　　　name
is 　　barrybarry@suse:~>
# my 與 name 中間用了 Tab 鍵取間隔，而在 name 的後方用了一個換行字符，所以 is 就
# 出現在第二行，接著於 is 與 barry 中間先用了 Tab 鍵再使用 Back Space 鍵，所以兩個
# 字串的間格是一個 Tab 鍵的間格再往前一個字元，最後則是使用 \c 來取消換行。


指令 alias

　設定指令別名的指令。藉由指令別名的設定，將可避免您於命令列上輸入過長的指令敘述。

指令語法：
alias 指令別名名稱＝'指令敘述'

範例說明：

barry@suse:~> alias barry="cat /etc/passwd /etc/fstab"
barry@suse:~> barry
# 這是在設定指令別名 barry，也就是說將來執行 barry 時，就等於是執行等號後方
# 一長串的指令敘述。另外別名的設定內容，使用單、雙引號皆可。

barry@suse:~> alias cp="cp -i"
# cp 這個複製指令的用法，在下一章會提到，先稍微講一下其基本語法：
# cp 複製來源 複製目的
# 當目的檔案已存在時，預設其內容會被覆蓋掉，但如果加上 "-i" 參數就可以避免這種
# 事情發生。因此我們可以為 cp 設上一個別名，也就是執行 cp 就等於執行 cp -i 之意，
# 這樣當目的檔已存在時，系統就會詢問您是否確定要覆蓋，如以下例子：
barry@suse:~> cp /etc/passwd file
cp: overwrite `file'?
# 當出現以上的訊息時，就知道 file 是一個已經存在的檔案，這時候可以輸入 n 來離開，
# 然後趕快去看一看 file 內容重不重要，再決定下一步要怎麼做。

barry@suse:~> \cp /etc/passwd file
# 於別名前加上 \ 之後，就等於是忽略這個別名的設定了。

barry@suse:~> unalias cp
# 欲取消別名的設定時，使用 unalias 指令就行了。

# 另外可以直接輸入 alias 指令，來查看目前系統有哪些別名：
barry@suse:~> alias
alias cd..='cd ..'
alias dir='ls -l'
alias l='ls -alF'
alias la='ls -la'
alias ll='ls -l'
alias ls='/bin/ls $LS_OPTIONS'
alias ls-l='ls -l'
alias md='mkdir -p'
alias o='less'
alias rd='rmdir'
# 以上這些別名的設定，可以在 /etc/bash.bashrc 的檔案中找到。

barry@suse:~> alias ll
alias ll='ls -l'
# 使用 alias 指令來檢視 ll 是否為別名。

# 再來也可以使用 type 指令來查看別名的資訊：
barry@suse:~> type -a ll
ll is aliased to `ls -l'

# 以下兩個指令的執行結果告訴我們 cp 指令目前並沒有設定別名：
barry@suse:~> type -a cp
cp is /bin/cp
barry@suse:~> alias cp
-bash: alias: cp: not found



3.7.2 變數的簡介

　所謂的變數，代表的就是會變動的資料值。當您正確的宣告一個變數後，系統便會在記憶體裡配置一個空間來存放這個資料值 (變數內容)，而將來即能以這個變數名稱來代表此記憶體位址所儲存的資料值。

　宣告變數的方式，在 "=" 左手邊為變數名稱，在 "=" 右手邊為變數內容 (變數值)，且等號本身的左右邊不能留有空格。另外變數名稱是由字母、數字及 _ 所組成，且名稱中的第一個字元不能是數字。

　而當您設定好一個變數之後，可以使用 "$" 這個符號來取得變數值，比如 var=test，那 $var 代表的就是要取得 var 的變數值 test，至於如何顯示所取得的變數值 ? 使用前面剛教過的 echo 指令就可以把變數內容做輸出了，如 "echo $var"。

範例說明：

barry@suse:~> my_name=barry
# 將 barry 這個值，設定給 my_name 這個變數。

barry@suse:~> echo $my_name
barry
# 輸出 my_name 的變數值。

barry@suse:~> my_name="barry chen"
# 當變數內容含有空白字元時，請用引號將其包住。

barry@suse:~> echo $my_name
barry chen

barry@suse:~> var="current time is `date`"
barry@suse:~> echo $var
current time is Thu Jan 11 03:26:13 CST 2007
# 如變數內容含有指令敘述，請記得使用反引號將其包住，這樣 shell 才會優先處理。

barry@suse:~> var='current time is `date`'
barry@suse:~> echo $var
current time is `date`
# 將變數內容整個用單引號包住時，會將此內容全部當成一般字元來處理。

barry@suse:~> echo \$var
$var
# 由於 $ 之前使用了跳脫字符，因此 $ 會變成一般字元，而失去取得變數值的功用。

barry@suse:~> var=/etc
barry@suse:~> var=$var/passwd
barry@suse:~> echo $var
/etc/passwd
# 上面的例子是說明變數具有繼承性 ( inherit )，它可以繼承同一個或另一個變數的值，
# 這個觀念很重要。

# 最後看看如何取消變數：
barry@suse:~> unset var　
# 這是移除 var 變數。

barry@suse:~> var=　
# 這是將 var 變數設定成空值 (null value)。


變數的輸出

　當您開啟一個 shell 的時候，可以藉著在提示號下輸入 "bash" 指令而進入到下一層 shell (sub-shell ) 來操作，而在下一層 shell 中還可以繼續進入到更深一層的 shell ﹔如果要回到上一層 shell，使用 Ctrl - D 或執行 exit 指令就行了。

　我們都知道 bash 本身就是一支可執行程式，所以當您進入到下一層 shell 的時候，就等於開啟另外一個子程式 (bash) 的意思。了解了這個觀念後，回到變數這裡，如果您先在這一層 shell 去宣告一個變數，然後使用剛剛才教過的方式進入到下一層 shell 後，會發覺這個變數無法提供給 sub-shell 的應用程式來使用，解決之道就是執行 export 指令來將變數做輸出，請看以下的範例說明：

barry@suse:~> var=/etc/passwd
barry@suse:~> echo $var
/etc/passwd

barry@suse:~> bash　
# 進入到下一層 shell。

barry@suse:~> echo $var

# 由於 var 變數無法提供給 subshell 中的 echo 程式來取用，所以無法顯示 var 的變數值。

barry@suse:~> exit
exit
# 回到上一層 shell。

barry@suse:~> export var
# 將剛剛宣告的變數使用 export 指令做輸出。

barry@suse:~> bash
# 再次進入到下一層 shell。

barry@suse:~> echo $var
/etc/passwd
# 這次就可以顯示 var 的變數內容了。
# 您也可以執行 "cat $var"，看看 cat 程式能否取用 var 變數值，並正確顯示檔案內容。

# 當然您也可以在宣告變數的同時，就將此變數做輸出：
barry@suse:~> export var="/etc/passwd"
　
　由以上的範例說明，各位可以了解到只要將變數使用 export 指令做輸出後，這個變數就可以提供給所有 sub-shell 環境中的應用程式來使用，所以可以稱其為使用者自訂的環境變數。

 　在剛剛的範例介紹都是以 bash 環境為主，若您使用的是 C shell，如 csh、tcsh 之類的，則必須使用 "set var=/etc/passwd" 這種方式來設定變數，若要讓此變數做輸出的話，則需使用 "setenv var /etc/passwd"，取消變數則是使用 "unset var" 或 "unsetenv var"。
$ 符號及反引號之補充說明

　在 RE 中，一般會把 pattern 用單引號 (single quote) 或雙引號 (double quote) 包住。一般來說，為 RE 加上單引號或雙引號其實是差不多的，但若嚴格區分的話，當使用雙引號時，裡邊有包含 「 $ 」( 此處的 $ 是位於變數名稱之前的 $，而非安置於 RE 最後面的 $ ) 及 「 ` 」(back quotes) 時，則其本身的特殊意義並不會改變，要是換成單引號，則他們會被當成一般字元來處理。

範例說明：

barry@suse:~> grep "^`echo barry`" /etc/passwd
barry:x:1004:100::/home/barry:/bin/bash

barry@suse:~> grep '^`echo barry`' /etc/passwd
# 搜尋 passwd 檔案中以 `echo barry` 做起始的那幾行，結果因為找不到，
# 當然就不會輸出任何內容。

barry@suse:~> grep "$USER" /etc/passwd
barry:x:1004:100::/home/barry:/bin/bash

barry@suse:~> grep '$USER' /etc/passwd
# 搜尋檔案裡頭包含 $USER 這個 pattern 的那幾行，結果當然是找不到囉。

3.7.3 環境變數 ( Environment Variable )

　 當一個使用者成功登入本機後，login 程式就會開啟一個 shell，以提供使用者的操作環境，所謂使用者的操作環境，就是在 shell 下的各項控制及設定，而每項環境的設定就是靠環境變數來達成。

　比如說在登入系統後，不是會看到提示號樣式嗎 ? 那這個提示號是怎麼來的，當然就是靠環境變數而來的啦 ! 又比如說您執行指令時，可以不需輸入這些指令的路徑而能直接執行，這也與環境變數有關。所以我們在登入系統的時候，會靠這些環境變數來將您個人環境建立起來，而平常在 shell 下所執行的一些程式，也才能取用這些環境變數的設定。

　shell 本身就提供相當多預設的環境變數，當然這些系統所提供的環境變數您可以去修改它，亦可自行去定義一些變數來使用。

　從變數的介紹一直到現在，大家可能已經了解到變數的種類了吧，分別是使用者自行定義的自訂變數，以及系統提供給我們的環境變數。自定變數可以選擇要不要使用 export 指令做輸出，如果沒做輸出時，則這一類的變數稱為 Local variable ﹔而系統提供的環境變數，則預設都是會做輸出的，不然不會稱其為環境變數 ( Environment Variable )。

　一般系統所提供的環境變數都是大寫的英文字母，以用來與使用者的自訂變數做個區分。一般常見的環境變數如 SHELL ( 使用者當前所使用的 shell )、PATH ( 路徑 )、HOME ( 家目錄 )、 PS1 ( 提示號 )、MAIL ( 使用者信件暫存處 )、USER ( 目前登入的使用者名稱 )、… 等等，若想看這些環境變數的內容，執行 "echo $變數名稱" 就行了。

　 查看目前存在於系統上的變數有哪些，可使用 set 或 env 指令。使用 set 指令所顯示的是所有的變數，包括環境變數及在這個 shell 下可使用的自訂變數 ﹔使用 env 指令所顯示的是所有的環境變數 ﹔另外 export 指令亦可顯示所有的環境變數。

　 接著底下就來看幾個環境變數。

HOME：定義使用者家目錄。

　平常執行 cd 指令時，可以切換至自己的家目錄，而您的家目錄到底在哪，就是根據 HOME 這個環境變數而來的啦。

barry@suse:~> echo $HOME
/home/barry

USER：定義使用者名稱。

　當使用 barry 的身分登入時，那 USER 就為 barry，使用 tina 的身分登入時，那 USER 當然就是 tina 了。

barry@suse:~> echo $USER
barry

SHELL：定義目前環境所使用的 shell 為何。

barry@suse:~> echo $SHELL
/bin/bash

PS1：定義提示號樣式。

　當使用者登入系統成功後，即可開始在提示號之後輸入指令，而此提示號樣式即根據 PS1 環境變數而來的 ﹔通常這也是一般所謂的第一提示號樣式。

barry@suse:~> echo $PS1
\u@\h:\w>

　看了以上 PS1 的內容是不是看不太懂，沒關係，以下這個表格就會解釋這個變數內容的意義：

命令 代　表　意　義
\u 顯示使用者名稱。
\h 顯示簡短的主機名稱。
\H 顯示完整的主機名稱。
\W 顯示目前工作目錄的位置，不過不是以完整路徑來顯示。
\w 顯示目前工作目錄的位置，以完整路徑來顯示。
\d 顯示日期。
\t 顯示 24小時制的時間。
\T 顯示 12小時制的時間。
\$ Uid 為 0 者 (root) 顯示 "#"，其它人則顯示 "$"。
\# 顯示目前所輸入的指令是第幾個。

　舉個例子，如果您還對 RedHat 的提示號樣式情有獨鍾的話，那麼可以馬上調整成跟它一樣：

barry@suse:~> PS1="[\u@\h \W]\\$ "
[barry@suse ~]$ _

PS2：定義第二提示號樣式。

　當您指令輸入得不完整卻又按下 Enter 鍵時，那麼在下一行一開始所看到的符號，即是第二提示號，這表示說您要繼續輸入未完成的部分才行。

barry@suse:~> echo $PS2
>
# 這是預設的第二提示號樣式。

barry@suse:~> echo "my linux distribution is :
> suse linux"
my linux distribution is :
suse linux
# 在這個範例中，由於字串開頭用了引號，但在結尾卻未加上引號而按下 Enter 鍵，
 # 那麼在下一行的開始，就會看到 ">" 這個第二提示號，表示說 shell 會等您為字串
 # 末端加上引號，這樣才算是一完整的指令。

# 現在調整一下第二提示號樣式：
barry@suse:~> PS2="C:\> "
barry@suse:~> cat /etc/termcap | head -n 200 | tail +100 \
C:\> | tee -a file
# 這樣應該對 PS2 有所了解了吧。

MAIL：定義使用者存放信件之處 ( mailbox )。

　環境變數 MAIL 所設定的變數內容，是定義使用者信件所存放的檔案位置，以提供給執行 mail 指令來收信時的一個參考路徑。

barry@suse:~> echo $MAIL
/var/mail/barry

PATH：定義指令的搜尋路徑。

　先來談一談在命令列上輸入指令時，為何不需要輸入這個指令的路徑而能直接執行 ? 其實 shell 在搜尋指令時是透過以下的程序：

  1. 如果指令是屬於 shell 的內建命令，則允許在任何工作目錄下直接執行這個指令，因為內建命令的程式碼，只是屬於整個 shell 程式的一部份。
  2. 如果指令非屬於 shell 內建命令，則會根據 PATH 中的目錄列表依序尋找，看看這些目錄下有無存在這個指令，如果有的話就允許直接執行，如果沒有的話，則您會看到 "command not found" 這類的訊息。/H

　所以將來您要是自己設計了一些程式，為了方便讓這些程式可以直接執行，可考慮將程式安置在 PATH 有定義的目錄中，不然也可以藉由修改 PATH 來達到您的目的。

Tips：環境變數 PATH 的內容，就是一個目錄列表，而每個目錄之間是使用 "：" 來做區隔。

barry@suse:~> echo $PATH
/home/barry/bin:/usr/local/bin:/usr/bin:/usr/X11R6/bin:/bin:/opt/gnome/bin:/opt/kde3/bin:
# 目前所看到的是一般使用者的 PATH 內容，其與 root 的 PATH 可是不同的：
barry@suse:~> su -
Password:　← 輸入 root 密碼。
suse:~ # echo $PATH
/sbin:/usr/sbin:/usr/local/sbin:/opt/gnome/sbin:/root/bin:/usr/local/bin:/usr/bin:/usr/X11R6/bin:/bin:/opt/gnome/bin:/opt/kde3/bin
# 看出兩者之間的差異了嗎。若要退回 user 的身分，按下 Ctrl - D 退出就行了。

# su 是用來變換身分的指令，當您使用 root 身分執行 "su - barry" 後，就可以馬上
# 變換成 barry 的身分，且過程之中不需要輸入 barry 的密碼。如是以一般使用者
# 的身分執行 "su -"，表示要切換成 root 的身分，此時會要求您輸入root 的密碼。

HISTFILE：定義指令紀錄檔位置。

barry@suse:~> echo $HISTFILE
/home/barry/.bash_history
# 這個檔案內所記載的指令紀錄，並不包括本次登入所輸入的指令，
# 一直要到您登出後才會記載進去。

HISTSIZE： 定義指令紀錄檔內所能儲存的最大指令數。

barry@suse:~> echo $HISTSIZE
1000
# 表示所能儲存的指令紀錄最多為 1000 筆。

PWD：顯示目前所在的工作目錄位置。

barry@suse:~> echo $PWD
/home/barry

# 您也可以直接輸入 pwd 指令來查看目前的工作目錄位置：
 barry@suse:~> pwd
/home/barry


3.7.4 環境設定檔
　
　修改系統預設環境變數的方式，除了可於命令列上重新定義外，還可透過環境設定檔來做調整，而兩者差異在於命令列上的修改，是屬於暫時性質的，於退出此 shell 後便消失，若您想讓此修改過後的環境變數永久有效的話，就必需在設定檔內做設定才行。

　當使用者成功登入系統時，馬上會取得一個 login shell ( 如果於此 shell 下再去啟用其他的 subshell，則稱為 non-login shell )，同時也取得了 shell 所提供的環境變數。您可以藉著修改一些環境設定檔來調整這些環境變數的內容。以下就是有關於環境設定檔的說明：

環境檔 相　關　說　明
/etc/profile 使用者在登入期間，首先會被 bash 所讀取的檔案。
~/.bash_profile 如果這個檔案存在，則在 bash 執行完 /etc/profile 後，會接著執行這個檔案的內容。
~/.bash_login 如果 ~/.bash_profile 不存在，則執行此檔。
~/.profile 如果 ~/.bash_profile 及 ~/.bash_login 都不存在，則執行此檔。SuSE 就是使用這個檔案。
~/.bashrc 此檔在登入期間也會被執行。
/etc/bash.bashrc 在載入 /etc/profile 的過程，也會執行 bash.bashrc 檔案。
suse:~ # cat /etc/profile
if test "$is" != ksh -a -r /etc/bash.bashrc ; then
　. /etc/bash.bashrc
fi

上面的內容是說，如果所使用的 shell 非 ksh，而且 /etc/bash.bashrc 是可讀的，那麼就會執行 /etc/bash.bashrc。
　　
　/etc/profile 及 /etc/bash.bashrc 的設定內容，可以套用在所有使用者的操作環境上，但只有 root 能去做設定，至於使用者則可以在自己家目錄下的 .profile 及 .bashrc 這兩個檔案做設定，以調整成自己想要的操作環境。不過在 root 的家目錄中，您會發現找不到這兩個隱藏檔，要解決的話很簡單，直接從 /etc/skel 複製過來即可：
suse:/etc/skel # cp .profile .bashrc ~

　了解了環境設定檔的大致狀況後，接著來談談進入 login shell 與 non-login shell 時，bash 所載入的檔案為何。當啟用 login shell 時，bash 會去讀取 /etc/profile 及 ~/.bash_profile、~/.bash_login、~/.profile 三者之一，而啟用 non-login shell 時，則只會讀取 ~/.bashrc 而已。

　使用者可以透過修改自己家目錄下的環境設定檔來製定自己的環境，比如可以把別名及函式設定在 ~/.bashrc 內，這樣這些別名及函式才能提供給 subshell (non-login shell) 的環境來使用。以別名來做例子好了，由於指令別名無法像變數一樣做輸出，所以當您把別名設定在 ~/.profile 時，會造成在 non-login shell 的環境裡無法引用這個別名，因為啟用 non-login shell 時並不會讀取 ~/.profile。

　以下就舉個實際的例子來印證：

# 先將指令別名設定在 ~/.profile 裡：
barry@suse:~> vi .profile
alias barry='ls -ld /home'

# 設定完成後，請先登出再登入，這樣 bash 才能重新執行 .profile 的內容。
# 不然也可以使用 source ( 或 . ) 指令來將檔案內容讀入目前的 shell 環境裡。
barry@suse:~> source .profile

# 接著測試指令別名 barry，應該可以正確執行噢：
 barry@suse:~> barry
drwxr-xr-x 8 root root 184 Jan 3 17:24 /home

# 現在進入到下一層 shell (non-login shell)：
barry@suse:~> bash

# 再執行一次指令別名：
barry@suse:~> barry
bash: barry: command not found
# 這時候就顯示錯誤訊息了，因為 ~/.profile 只有在啟用 login shell 時才會被執行。

　現在將指令別名改設定在 ~/.bashrc：
barry@suse:~> vi .bashrc
alias mary='ls -ld /var'

barry@suse:~> source .bashrc

# 接著作測試：
 barry@suse:~> mary
drwxr-xr-x 16 root root 392 Oct 7 23:08 /var

# 進入到下一層 shell：
barry@suse:~> bash

# 再執行一次指令別名： 
barry@suse:~> mary
drwxr-xr-x 16 root root 392 Oct 7 23:08 /var
# 還是可以正確被執行，因為啟用 non-login shell 時會執行 ~/.bashrc。

PATH 的設定範例

　以下舉個環境變數 PATH 的實際設定例子好了。假使使用者 barry 在 /tmp 底下建立一個名為 test.sh 的 script，其內容如下：

barry@suse:~> vi /tmp/test.sh
#! /bin/bash
echo "I am $USER."
echo "This is a script for test."

# 接著讓 test.sh 具有可執行的權限：
barry@suse:~> chmod +x /tmp/test.sh

　當 barry 直接執行此 script 時，會出現如下訊息：

barry@suse:~> test.sh
bash: test.sh: command not found

　各位應該知道產生這個錯誤訊息的原因吧，因為 test.sh 並不是 shell 的內建命令，而此 script 所在的目錄 /tmp，也不存在於 PATH 變數內容中，所以當然會出現找不到指令的訊息囉。此時您可以藉著指出 test.sh 的路徑來解決這個問題：

# 使用相對路徑的方式來執行：
barry@suse:/tmp> ./test.sh
I am barry.
This is a script for test.

# 使用絕對路徑的方式來執行：
barry@suse:~> /tmp/test.sh
I am barry.
This is a script for test.

　現在想讓 barry 不管在任何工作目錄下都只需輸入 test.sh 就能執行，則必須修改環境變數 PATH ：

barry@suse:~> PATH="$PATH:/tmp"
# 前面提過的變數繼承性概念，應該還不健忘吧。

# 確認一下 /tmp 是否已存在 PATH 中：
barry@suse:~> echo $PATH
/home/barry/bin:/usr/local/bin:/usr/bin:/usr/X11R6/bin:/bin:/opt/gnome/bin:/opt/kde3/bin:/usr/lib/mit/bin:/usr/lib/mit/sbin:/tmp

# 最後測試看看吧：
barry@suse:~> test.sh　
My name is barry.
This is a script for test. 　

　以上在命令列上所做的修改，於結束這個 shell 之後 (logout) 就失效了。如想做永久設定的話，則請修改個人的環境設定檔：

barry@suse:~> vi .profile
PATH="$PATH:/tmp"
export PATH

　存檔離開後，請先登出再登入，或者直接執行 source，完成後您再自行測試看看。

　今天如果管理者寫了一支程式，希望能讓所有使用者都能直接執行它，那您可以將這支程式安置在使用者 PATH 裡頭有定義的那些目錄中的其中一個，而且建議最好是放在 /usr/local/bin 裡面，因為這樣也方便您管理自己所設計的程式。那如果程式所存在的目錄不包含在 PATH 中的話，就只好到 /etc/profile 裡去修改 PATH 囉。另外一般使用者自己所寫的程式，可以放在 ~/bin 之中，因為 PATH 裡就包含了這個目錄。

其他環境的調整

　不曉得各位有沒有注意到，當您使用 root 身分執行 ls 指令時，居然會顯示隱藏檔，但使用一般 user 的身分去執行卻又不會，這是因為 bash.bashrc 裡頭設定的關係，稍微修改一下就行了：
suse:~ # vi /etc/bash.bashrc
if test "$UID" = 0 ; then
　LS_OPTIONS="-A -N $LS_OPTIONS -T 0"
else
　LS_OPTIONS="-N $LS_OPTIONS -T 0"
fi
# 將上頭那個 -A 參數移除即可。

　另外還有個預設語系編碼的問題，如果當初您安裝 SuSE 時，預設語系是選擇繁體中文，那麼當您使用一般 user 的身分登入純文字介面後，會發現在操作的過程有部份字元會顯示不出來，就像這個樣子：
barry@suse:~> cat nofile
cat: nofile: ?????????

　此時您可以在 /etc/profile 的最後頭補上一段敘述就可以搞定：
suse:~ # vi /etc/profile
if [ -z $DISPLAY ] ; then
　LANG=POSIX
fi

　還有一種狀況是當以一般 user 身分登入 X-WINDOW 時，桌面環境及選單的字體可以正確顯示中文，但換成 root 登入時卻顯示英文，針對這個問題的解決，請修改 /etc/sysconfig/language：
suse:~ # vi /etc/sysconfig/language
ROOT_USES_LANG="yes" ← 由 ctype 改成 yes。

suse:~ # SuSEconfig
# 讓新的設定值生效。


3.7.5 指令紀錄

　在 3.7.3 節有提到過指令紀錄檔，而這裡則是要使用 history 指令去叫出之前所輸入過的指令 ﹔指令紀錄檔內是不包含本次登入所輸入的指令，但 history 則是有包括在內。

範例說明：
barry@suse:~> history
# 顯示輸入過的指令敘述，含本次登入在內。

barry@suse:~> history 6
# 顯示最後面六個指令敘述。

barry@suse:~> !150
# 表示執行 history 中編號 150 的指令 。

barry@suse:~> !cat
# 表示執行 history 中最後一個 cat 指令。

barry@suse:~> !!
# 執行前一個指令。


3.8 擅用 manual page

3.8.1 man 指令操作

　當您在執行指令或設定檔案時，往往會想要了解這個指令的詳細用法或檔案的設定格式，這時候使用系統提供給我們的 man page ( Manual page) 就對啦，而查詢 man page 所使用的指令就是 man。

指令語法：
man [options] [section] name

　先來談談 section 的部分。在 /usr/share/man 目錄下可以看到有 man1 ~ man9 的目錄，而每個目錄是分別存放著不同性質的 man page，比方說 man1 是存放一般使用者常使用指令的 man page，這就是 section1，所以接下來就來看看每個 section 所存放的 man page 為何 ?

SECTION 說　明　文　件　性　質
section 1 存放一般使用者所執行指令的 man page。
section 2 核心提供的系統呼叫函式之相關說明。
section 3 函式庫裡的函式相關說明。
section 4 特殊裝置檔案說明。
section 5 一般檔案的設定格式說明。
section 6 遊戲說明。
section 7 聚集套件協定之說明。
section 8 管理者所使用指令的說明。

　每一個 section 的詳細說明可 "man section intro" 來查閱，比如 "man 5 intro"。

　另外當您在使用 man 指令時，如未指定要查詢的 section 為何，則其預設的搜尋順序為：
　man1 → man8 → man2 → man3 → man4 → man5 → man6 → man7

參數說明：
-k 搜尋 manual page 中的名稱或簡短描述裡，含有 -k 參數其後所指定的關鍵字。
"man -k name" 同等於 "apropos name"。
-f 搜尋 manual page 中的名稱為 -f 參數後所指定的關鍵字。
"man -f name" 同等於 "whatis name"。

範例說明：
suse:~ # man passwd
# 此時您看到的會是指令 passwd 的 man page，這就是預設搜尋順序的關係。

suse:~ # man 5 passwd
# 指定查詢 section 5 的 man page，所以您看到的是檔案 passwd 的 man page。

suse:~ # man -k xargs
xargs (1)　　　　- build and execute command lines from standard input
xargs (1p)　　　 - construct argument lists and invoke utility
 # 也可以執行 "apropos xargs"，看看結果有沒有一樣。

suse:~ # man -f grep
grep (1)　　　　 - print lines matching a pattern
grep (1p)　　　　- search a file for a pattern
 # 試試看 "whatis grep" 的結果如何。

　再來還有個 whereis 指令，其可用來查詢程式或檔案所在的位置及其 man page 所存放之處：

suse:~ # whereis ls
ls: /bin/ls /usr/share/man/man1/ls.1.gz /usr/share/man/man1p/ls.1p.gz


進入 man page 之後的操作

　在進入 man page 後，看您要使用 [Page Up]、[Page Down] 來快速翻頁，或使用上下鍵來一行一行捲動都行。另外 [Home] 按鍵可以讓您回到首頁，[End] 按鍵則是直接到末頁。

　進入 man page 後，往往裡頭所呈現的資料都非常可觀，因此擅用搜尋技巧就很重要了。以下即列出如何搜尋關鍵字的位置：

所做輸入 代　表　意　義
/pattern 從目前所在的那一頁開始向下搜尋。
n 找下一個 pattern 的位置。
N 回上一個 pattern 的位置。
? pattern 從目前所在的那一頁開始向上搜尋。
q 退出 man page。

3.8.2 man 搜尋 man page 的方式

　當使用 man 做查詢時，其實系統是根據 MANPATH 變數所定義的目錄或 /etc/manpath.config 檔案去尋找 man page。如果  MANPATH 變數存在的話，就會忽略 /etc/manpath.config 的設定，那當然 MANPATH 變數不存在時，就以 /etc/manpath.config 為主囉 ! 您可以在 /etc/profile 中找到 MANPATH：

suse:~ # vi /etc/profile
MANPATH=`test -x /usr/bin/manpath && /usr/bin/manpath -q`
# 變數內容是說，如果 manpath 程式可執行的話，就執行 manpath -q。
 # 請自己試看看執行 manpath -q 的結果為何。
export MANPATH
　
　 如不想使用 MANPATH 變數，而是希望以 /etc/manpath.config 為主，則請將 MANPATH 相關設定註解起來，此時只要在每一行的最前頭補上個 # 符號就行了。

　 假使現在有個軟體的 man page 是安置在 /usr/local/proftpd/man 目錄之中，而為了要讓 man 指令找得到這些文件，因此必須要把這個目錄設定在 MANPATH 變數或 /etc/manpath.config 檔案之中，以下分別來看看要怎麼做：

設定在檔案裡：
suse:~ # vi /etc/manpath.config
MANDATORY_MANPATH　　　　 /usr/share/man
MANDATORY_MANPATH 　　　　/usr/share/man/allman
MANDATORY_MANPATH 　　　　/usr/X11R6/man
MANDATORY_MANPATH 　　　　/usr/local/man
MANDATORY_MANPATH 　　　　/opt/man
MANDATORY_MANPATH 　　　　/usr/local/proftpd/man 　← 自行補上這一行。

設定在 MANPATH 變數中：
suse:~ # vi /etc/profile
MANPATH=`test -x /usr/bin/manpath && /usr/bin/manpath -q`:/usr/local/proftpd/man
export MANPATH 　　　　　　　　　　　　　　　　　　　　　↑
　　　　　　　　　　　　　　　　　　　　　　　　　　　 補在後端即可。
suse:~ # source /etc/profile

　除了 man page 外，像 /usr/share/doc 裡頭也存放著很多的說明文件，比如與套件相關的說明、常見問答集 FAQ、HOWTO 文件等等。裡邊內容很豐富噢，去那逛一逛或許會有意想不到的收穫噢。
